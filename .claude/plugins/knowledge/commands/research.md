---
description: Запустить sci-researcher (Opus) — глубокое Q&A по базе знаний с цитированием источников.
---

Проведи исследование по базе знаний и ответь на вопрос.

Входные данные: $ARGUMENTS — вопрос для исследования. Поддерживаемые флаги:
- `--save` — сохранить ответ как новую wiki-страницу в `knowledge/wiki/qa/{YYYY-MM-DD}-{slug}.md` (компаундинг). По умолчанию **не сохраняет**, чтобы wiki не замусоривалась.
- `--topic <name>` — ограничить поиск папкой темы.

## Алгоритм

1. **Проверь аргументы**
   Если $ARGUMENTS пустой:
   > Укажи вопрос: `/knowledge:research <твой вопрос> [--save]`

   Извлеки флаги (`--save`, `--topic`) перед передачей вопроса агенту.

2. **Изучи базу знаний**
   Прочитай `knowledge/wiki/index.md` (или `wiki-llm/index.md` если есть Уровень 3).

3. **Найди релевантный контент**
   - Статьи в `knowledge/wiki/` по теме вопроса (Grep по тегам / заголовкам)
   - Связанные транскрипты в `knowledge/raw/videos/`
   - Составь список релевантных файлов для передачи агенту

4. **Вызови sci-researcher**
   ```
   Agent(subagent_type: "sci-researcher",
         prompt: "Вопрос: <question>. Релевантные файлы: <list>.")
   ```

5. **Вывод агента**
   - Чёткий ответ на вопрос
   - Ключевые концепции
   - Цитаты из источников с таймкодами (если из видео)
   - Связанные статьи для дальнейшего изучения
   - Открытые вопросы

6. **Save (только если `--save` И ответ содержит ≥3 cited-факта)**:
   - slug — из вопроса, kebab-case, ≤ 50 символов.
   - путь: `knowledge/wiki/qa/{YYYY-MM-DD}-{slug}.md`.
   - frontmatter:
     ```yaml
     ---
     title: <вопрос>
     type: qa
     tags: [qa, <темы из найденных статей>]
     sources_internal: [[стр-1]], [[стр-2]]
     date_created: YYYY-MM-DD
     status: draft
     ---
     ```
   - тело — отполированный ответ + секция `## Связи` со всеми `[[wikilinks]]`.
   - предложи `/knowledge:library apply` чтобы внести в `index.md`.

7. **Append log.md** (всегда, независимо от `--save`):
   ```markdown
   ## [YYYY-MM-DD] research | <вопрос>  save=<yes/no>
   - Цитировано статей: N
   - Пробелы: <список или "нет">
   - Сохранено как: [[path]]  (если --save)
   ```

## Примеры использования

```
/knowledge:research что такое метод Карпати для базы знаний
/knowledge:research как работает faster-whisper
/knowledge:research чем отличается curator от synthesizer
/knowledge:research экономия токенов в Claude Code --topic claude-code --save
```

## Если в базе знаний нет ответа

Агент честно сообщит:
> Тема "{вопрос}" пока не покрыта в wiki. Предлагаю:
> - `/knowledge:transcribe <url>` — добавить видео-источник
> - `/knowledge:curate <url>` — добавить статью

`--save` в таком случае игнорируется (нечего сохранять без cited-фактов).
