Ответь на вопрос пользователя, опираясь на его ЛИЧНУЮ БАЗУ ЗНАНИЙ (метод Карпати:
`wiki/` — дистиллированные статьи, `raw/` — исходники: транскрипты видео, книги, статьи).

Вопрос: **$ARGUMENTS**

## Где база знаний (KB)

Определи корень KB по порядку — первый сработавший (НЕ угадывай абсолютный путь):

```bash
KB="${KOS_KNOWLEDGE_ROOT:-}"                                   # 1. env (канон)
[ -z "$KB" ] && [ -f ~/.kos/launcher.json ] && \
  KB=$(python3 -c "import json,os;print(json.load(open(os.path.expanduser('~/.kos/launcher.json'))).get('KOS_KNOWLEDGE_ROOT',''))" 2>/dev/null)  # 2. ~/.kos/launcher.json
[ -z "$KB" ] && [ -d ../knowledge-base/wiki ] && KB=$(cd ../knowledge-base && pwd)  # 3. соседний репо
echo "${KB:-NOT_FOUND}"
```

Если `NOT_FOUND` — скажи пользователю задать `KOS_KNOWLEDGE_ROOT` (env или `~/.kos/launcher.json`),
НЕ подставляй путь наугад. Команда НЕ зависит от текущего проекта — всегда смотрит в KB.
qex держит индексы в одном глобальном сторе (`~/.qex/projects/`), ключ — путь: индекс базы
строится один раз и переиспользуется из любого проекта; код текущего проекта и база знаний —
разные изолированные индексы.

## Алгоритм

1. **Разбери флаги** из $ARGUMENTS:
   - `--raw` — искать ещё и в `KB/raw/` (книги/видео/статьи дословно), не только в `wiki/`.
   - `--limit N` — сколько результатов (по умолчанию 8).
   Остаток строки — сам вопрос.

2. **Поиск через qex** (primary). `mcp__qex__search_code`:
   - `path = "<KB>/wiki"`, `query = <вопрос>`, `limit = N`.
   - Если `--raw` — ВТОРОЙ вызов с `path = "<KB>/raw"`, объедини результаты.
   - qex сам индексирует путь при первом обращении и сам падает на BM25 без эмбеддингов — это нормально.

3. **Fallback, если qex недоступен** (нет MCP-инструмента / ошибка): `Grep` по `<KB>/wiki/`
   (и `<KB>/raw/` при `--raw`), затем `Read` 2-4 самых релевантных файла. Не передавай задачу «впустую».

4. **Синтезируй ответ:**
   - Чёткий ответ своими словами.
   - Ключевые факты/концепции из найденного.
   - **Цитирование** — кликабельными ссылками на файлы, напр. `[raw-wiki-pipeline.md](<KB>/wiki/concepts/raw-wiki-pipeline.md)` (подставь резолвнутый `$KB`). Для видео — таймкоды, если есть.
   - Связанные статьи (`[[wikilinks]]` из frontmatter) для углубления.

5. **Если ответа в базе нет** — честно:
   > Тема «<вопрос>» пока не покрыта. Добавить: `/knowledge:kb-add <url|файл>` (когда настроено) или закинь в `KB/inbox/` и скурируй дома.

## Принципы
- Это **чтение** базы из любого проекта. Ничего в базе не меняй.
- `wiki/` — приоритет (дистиллят, высокий сигнал). `raw/` — по `--raw` или для дословной цитаты/детали.
- Отвечай на русском (если не попросили иначе). Идентификаторы/пути — как есть.

## Примеры
```
/knowledge:kb что такое метод Карпати
/knowledge:kb как работает faster-whisper --raw
/knowledge:kb чем curator отличается от synthesizer --limit 5
```
