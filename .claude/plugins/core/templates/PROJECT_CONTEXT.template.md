# Project Context Registry

Сводный реестр per-module знаний проекта. Генерируется скриптом
`scripts/aggregate_context` из per-module `CONTEXT.md` и `DECISIONS.md`.
Содержимое между маркерами — auto-generated, **руками не править**.
Текст вне маркеров — рукописный, сохраняется при пересборке.

**Запуск:** `python -m scripts.aggregate_context` (write) /
`--check` (CI gate).

**Конвенции:**
- `<module>/CONTEXT.md` — лёгкий per-module документ (Purpose / Gotchas / …)
- `<module>/DECISIONS.md` — формальные ADR (нумерованные `ADR-{CODE}-NNN`)
- Глобальные cross-module ADR — в `docs/decisions/NNNN-*.md`
  (создаются через `/dev:adr`)

## Как добавить новый модуль с CONTEXT.md

1. Создай `<package>/<module>/CONTEXT.md` из шаблона
   `.claude/plugins/core/templates/CONTEXT.template.md`.
2. Заполни хотя бы `Purpose`. Остальные секции — по мере накопления.
3. (Опц.) Если у модуля будут формальные ADR — создай ещё
   `<package>/<module>/DECISIONS.md` из `.claude/plugins/core/templates/DECISIONS.template.md`
   с уникальным `module_code:` в frontmatter.
4. Запусти `python -m scripts.aggregate_context` — реестр обновится.
5. Закомить и `CONTEXT.md`/`DECISIONS.md`, и обновлённый `PROJECT_CONTEXT.md`.

## Как добавить cross-module (глобальный) ADR

Используй `/dev:adr <title>` — создаёт `docs/decisions/NNNN-<slug>.md`.
Эти ADR имеют формат `ADR-NNN` (без module-кода) и не индексируются
скриптом `aggregate_context` — у них своя нумерация и место хранения.
