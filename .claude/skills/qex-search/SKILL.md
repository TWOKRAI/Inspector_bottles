Выполни семантический поиск по кодовой базе через qex MCP перед рефакторингом или изменением логики.

## Алгоритм поиска

### 1. Предварительные условия
Убедись, что qex доступен:
```bash
# Холодный старт (если нужно)
docker start qdrant
ollama serve &
```
Проверь статус: `mcp__qex__get_indexing_status()`

Если индекс пустой или устаревший: `mcp__qex__index_codebase(force=True)`

### 2. Поиск
Вызови `mcp__qex__search_code(query="<запрос на русском или английском>")`.

**Хорошие запросы:**
- По назначению: `"обработка входящих сообщений процесса"`, `"регистрация модуля в лаунчере"`
- По паттерну: `"Dict at Boundary to_dict from_dict"`, `"FieldRouting channel подписка"`
- По ошибке: `"ModuleNotFoundError импорт"`, `"утечка памяти QObject"`

**Плохие запросы:** конкретные имена файлов (используй Glob/Grep), синтаксис Python (используй Grep).

### 3. Интерпретация результатов
- Результаты отранжированы по релевантности (BM25 + эмбеддинги).
- Проверь топ-3–5 результатов через `Read`.
- Если нашёл нужный файл — запусти дополнительный Grep для точной навигации.

### 4. Когда использовать qex-search
- Перед рефакторингом любого модуля
- При поиске всех мест использования паттерна
- Когда точное имя класса/функции неизвестно
- При анализе влияния изменений

### 5. Когда НЕ использовать
- Известен точный путь к файлу → `Read`
- Известно точное имя класса/функции → `Grep`
- Поиск по синтаксису → `Grep` с regex
