**ОБНОВЛЕНО 2026-07-21** (перепроверено чтением `scripts/validate_commit/validate_commit.py`
на ветке `fix/backend-ctl-proof-discipline`, Task 1.4): старая версия этой памяти
(«перенос строки ломает парсер, hook падает с Missing required trailers») —
УСТАРЕЛА. Баг был реальным 2026-07-13 (задача G.6), но с тех пор `parse_message`
переписан: `TRAILER_RE`-непоматчившая строка внутри уже начавшегося trailer-блока
теперь фолдится как ПРОДОЛЖЕНИЕ значения предыдущего трейлера (код +
явный комментарий об этом прямо в `parse_message`, ссылающийся на этот же трап).
Перенос `Why:`/`Layer:`/`Tested:` на вторую строку больше НЕ роняет коммит.

**Why:** тем не менее, однострочные трейлеры остаются рекомендацией (сам код
это прямо называет «остаётся рекомендацией») — короче для `git log --grep`/
`git log --trailer=`, не зависит от точности фолдинга при будущих правках
парсера. Держать одной строкой — всё ещё хорошая практика, просто это больше
не hard requirement.

**How to apply:** предпочитай `Why:`/`Layer:`/`Refs:`/`Tested:`/`Risk:`/
`Reversible:`/`Rejected:` одной физической строкой (даже длинной) — но если
перенеслось непреднамеренно, коммит НЕ будет отклонён. Перед `git commit -F`
можно быстро проверить парсером: `python3 -c "import sys;
sys.path.insert(0,'scripts/validate_commit'); import validate_commit as v;
print(v.validate(open('msg.txt').read()).ok)"` — но это уже подстраховка, не
обязательный шаг.

См. также [[project_commit_format]] (обязательные трейлеры Why/Layer, общее
описание hook'а).
