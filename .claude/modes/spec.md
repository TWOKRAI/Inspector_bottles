# Spec Mode — Живое продуктовое ТЗ

Загружается при задачах: создать/обновить `docs/direction/*.md`, описать приложение с точки зрения пользователя, синхронизировать ТЗ с кодом.

## Философия

**Живое ТЗ** — это продуктовая спецификация, которую пользователь редактирует как инструкцию для Claude. Не техническая документация кода, а описание поведения приложения глазами пользователя.

Цикл:
```
spec-writer создаёт/обновляет docs/direction/*.md
↓
Пользователь читает и правит (это пользовательский интерфейс к коду)
↓
/spec-sync → manager сравнивает spec vs код → Task X.Y в plans/
↓
/pipeline или /implement → код приходит в соответствие с spec
```

## Агенты

| Агент | Skill | Назначение |
|-------|-------|-----------|
| **spec-writer** (Sonnet 4.6) | `/spec` | Создать / обновить `docs/direction/` |
| **manager** (Sonnet 4.6) | `/spec-sync` | Сравнить spec с кодом → декомпозировать → Task X.Y |

## Структура `docs/direction/`

Для каждого приложения в `apps/`:

```
apps/<app>/docs/direction/
  00_INDEX.md         — Оглавление, версия, инструкция
  01_layout.md        — Главное окно: меню, тулбары, панели
  02_<tab1>.md        — Основная вкладка / экран
  03_<tab2>.md        — Вторая вкладка
  04_<panel>.md       — Боковые панели (опционально)
  05_<flow>.md        — Ключевые сценарии (опционально)
```

Каждый файл — YAML frontmatter + markdown. Описывает ЧТО видит пользователь и КАК взаимодействует, а не как реализовано внутри.

## Типовые сценарии

```
Новое приложение, продумать UX      →  /spec <app>  → spec-writer создаёт skeleton
Пользователь хочет изменить UX      →  правит spec  →  /spec-sync <app>  →  manager → Task
Синхронизация после большого рефа  →  /spec-sync <app>  → manager обновляет spec
Понять, что должно быть в apps/X   →  прочитать apps/X/docs/direction/00_INDEX.md
```

## Правила

1. **Spec — source of truth для UX** — при конфликте spec vs код, spec побеждает (если пользователь явно не сказал обратное)
2. **Никакой технической реализации в spec** — классы, функции, БД-схемы не упоминать. Только поведение
3. **Язык русский** — как и весь проект
4. **Versioning через 00_INDEX.md** — каждое обновление меняет `version:` в frontmatter и добавляет changelog
5. **Spec живёт рядом с приложением** — `apps/<app>/docs/direction/`, не в корне `docs/`
