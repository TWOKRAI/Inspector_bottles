Ручной захват урока **в момент verified-перехода** (red→green, фикс подтверждён,
решение принято) или **на границе Task** (X.Y done → X.Y+1 в multi-phase плане).
Один урок = один файл. Это основной триггер capture rail — не жди
`/core:team:wrap-up` (см. `dev.md` → "Memory discipline (capture at Task boundary)").

## Аргументы

`$ARGUMENTS` — суть урока одной фразой (опц.). Если пусто — выведи урок из
контекста последнего verified-перехода в сессии.

## Гейт (не пиши, если не проходит)

Захватывай только при срабатывании WHEN-гейта из `.claude/CLAUDE.md` → Memory
(OVERRIDE) → «Когда захватывать»:

- фикс занял больше одной попытки (root cause был неочевиден);
- повторно наступил на те же грабли;
- пользователь дал правило / поправку / предпочтение;
- принято нетривиальное решение, которого нет в коде, git-логе или плане.

Не проходит → скажи «нечего захватывать» и остановись. **Не выдумывай урок**
и не записывай то, что уже хранят код / git / план / `CLAUDE.md` (FORBID).

## Шаги

1. **Сформулируй урок** одной фразой (она же — ключ для дедупа) и выбери тип:
   `feedback` (как мне работать) · `project` (текущая работа/решение) ·
   `user` (кто пользователь) · `reference` (внешний указатель).

2. **Дедуп — grep по КАЖДОМУ ключу отдельно** перед любой записью. Целая фраза
   почти никогда не совпадёт (один quoted-паттерн = непрерывная подстрока) — ищи
   по отдельным словам урока И 1-2 синонимам через `-e`, целясь в строки
   `description:` существующих записей:
   ```bash
   grep -rinl --include='*.md' -e 'слово1' -e 'синоним1' -e 'слово2' .claude/memory/ 2>/dev/null
   ```
   Хоть один хит → открой файл, реши near-match: да → **UPDATE** (допиши нюанс /
   повтор / эскалацию), поправь строку в `MEMORY.md`, **на этом стоп** (дубли не
   плодим). Ноль хитов по **всем** вариантам → новый файл (шаг 3).

3. **Нет совпадения → новый файл** `.claude/memory/<kebab-slug>.md` в формате Карпатия:
   ```markdown
   ---
   name: <kebab-slug>
   description: <одна строка — по ней идёт recall>
   metadata:
     type: feedback | project | user | reference
   ---

   <факт одним абзацем.>

   **Why:** <почему важно / откуда взялось — дата, триггер>
   **How to apply:** <как применять в следующий раз; для feedback/project обязательно>

   Связи: [[related-memory-slug]]
   ```
   Относительные даты → абсолютные («сегодня» → YYYY-MM-DD).

4. **Индекс.** Добавь одну строку в нужную секцию `.claude/memory/MEMORY.md`:
   ```
   - [Title](slug.md) — <короткий хук>
   ```
   Индекс ≤200 строк, одна строка на запись.

5. **Подтверди** одной строкой: create или update, путь файла, тип.

## Не делать

- Не дублировать то, что уже в коде / git / плане / `CLAUDE.md`.
- Не создавать новый файл при наличии near-match — обновляй существующий.
- Не писать записи «на будущее» без сработавшего гейта.
- Не трогать чужую память из других проектов.
