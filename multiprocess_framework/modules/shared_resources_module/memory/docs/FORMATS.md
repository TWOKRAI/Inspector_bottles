# Режимы pack/unpack — скорость и ограничения

## Сравнение режимов

| Режим | Запись | Чтение | Оценка скорости | Риски |
|-------|--------|--------|-----------------|------|
| **pack_fast=True** (по умолчанию) | np.copyto | — | ~2× быстрее записи | Нет |
| **pack_fast=False** (legacy) | tobytes | — | Базовая | Нет |
| **copy=True** (по умолчанию) | — | .copy() | Базовая | Нет |
| **copy=False** | — | view | ~2× быстрее чтения | Данные живут до следующей записи в слот |

## Оценка прироста скорости

Для кадра 480×640×3 uint8 (~921 KB):

| Операция | Legacy | Fast | Прирост |
|----------|--------|------|---------|
| Запись (pack) | ~2–3 ms | ~0.5–1 ms | **~2–3×** |
| Чтение (unpack) | ~1–2 ms | ~0.3–0.5 ms (copy=False) | **~2–4×** |

*Замеры зависят от CPU, кэша, ОС. Порядок величин корректен.*

## Детали

### pack_images_fast (запись)

- **Как:** `np.copyto(dest, img)` — один memcpy из ndarray в буфер.
- **Было (legacy):** `img.tobytes()` → копия в bytes → присваивание в buffer (ещё копия).
- **Плюсы:** Меньше аллокаций, меньше копирований, выше скорость.
- **Минусы:** Padding не заполняется нулями (слот перезаписывается при следующей записи).
- **Совместимость:** Результат идентичен legacy для заголовка и данных изображения.

### unpack_images(copy=False) (чтение)

- **Как:** Возвращает `np.frombuffer(...).reshape()` без `.copy()`.
- **Плюсы:** Нет копирования, быстрее.
- **Минусы:** View указывает на SharedMemory. При следующей записи в этот слот данные станут невалидны.
- **Когда использовать:** Когда данные обрабатываются сразу и не хранятся после чтения.

### unpack_images(copy=True) (чтение, по умолчанию)

- **Как:** `.copy()` — данные копируются в новый ndarray.
- **Плюсы:** Безопасно, данные остаются валидными после записи в слот.
- **Минусы:** Дополнительное копирование.

## Рекомендации

1. **По умолчанию:** `pack_fast=True`, `copy=True` — быстрая запись, безопасное чтение.
2. **Максимальная скорость:** `pack_fast=True`, `copy=False` — если данные используются сразу и не сохраняются.
3. **Совместимость:** `pack_fast=False` — если нужна полная совместимость со старым форматом (включая padding).
