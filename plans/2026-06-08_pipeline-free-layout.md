# План: свободное размещение нод графа Pipeline + персист позиций/фиксации

**Ветка:** `feat/pipeline-free-layout` · **Slug:** `pipeline-free-layout`
**Статус:** черновик (реализация — после закрытия хвостов на `feat/line-filter-virtual`)

## Контекст (зачем)

Владелец о визуальном графе вкладки Pipeline:
1. Не нравится, что ноды **автоматически выравниваются в линию** — хочет свободное перемещение.
2. Нужна **возможность фиксировать** ноды (есть, но session-only).
3. **Позиции должны храниться в рецепте** → при перезапуске положение не меняется.
4. На старте граф **отображается криво**, приходится жать «Раскладку» → хочет, чтобы позиции грузились из рецепта и сразу было «как надо».
5. При перетаскивании ноды **объединяются под один процесс** — так быть НЕ должно. Менять процесс/воркер — **только через combo** в инспекторе.

## Корень проблемы (расследование кода, HIGH confidence)

- **Свободно подвинуть ноду сейчас НЕЛЬЗЯ.** `GraphScene.on_node_drag_finished` (`graph_scene.py:243-274`) по центру ноды определяет процесс-контейнер и шлёт `plugin_drop_requested` → `presenter.on_plugin_dropped` (`presenter.py:378-423`): если процесс/индекс не изменились → **snap-back (reload)**; если центр попал в чужой контейнер → `MovePlugin` (**«объединение под процесс»**). Отсюда и п.1, и п.5, и принуждение к «Раскладке».
- **Авто-layout на старте НЕ вызывается** (`tab.py:308-311`) — это уже корректно. Ноды берут позицию из `_gui_positions` (`presenter._node_position` `presenter.py:1379-1404`): `node_id → process_name → дефолтный кластер`.
- **«Криво на старте»** = рецепт `line_filter_inspect.yaml` (и др.) **не содержит** `metadata.gui_positions` → дефолтный кластер.
- **Фиксация** (`_locked_nodes: set[str]` `presenter.py:136`) — session-only: ставит `ItemIsMovable=False` + золотая рамка + пропуск в авто-layout. В рецепт НЕ сохраняется.
- **Сохранение позиций есть, но не авто:** `export_topology_with_positions` (`presenter.py:541-552`) пишет `metadata.gui_positions` из `scene.get_all_node_positions()`. `load_topology_from_config` (`presenter.py:506-531`) читает их. Sync при мутациях — `presenter.py:1016-1017`.
- **Модель:** `node_id = {process}.{plugin}`; ноды — `NodeItem` (Movable/Selectable), фон-рамка процесса — `ProcessContainerItem` (Z=-1, fit_to_members по членству, не по позиции).

## Задачи

### Task 1 — Drag = только перемещение (НЕ смена процесса)
**Цель:** перетаскивание ноды меняет ТОЛЬКО её позицию; членство в процессе не трогается.
**Файлы:** `graph/graph_scene.py`, `presenter.py`.
**Шаги:**
1. `on_node_drag_finished`: убрать определение `to_process` по контейнеру и эмит `plugin_drop_requested` для смены процесса. Вместо этого — зафиксировать новую позицию ноды (обновить `_gui_positions[node_id]`), без reload/snap-back.
2. Reorder внутри процесса по drag — тоже убрать (порядок плагинов меняется не drag'ом; если нужно — отдельной кнопкой/combo, вне scope).
3. Смена процесса/воркера остаётся ТОЛЬКО через combo инспектора (`move_to_process_requested` / `assigned_worker` — уже есть, не трогаем).
**Acceptance:**
- [ ] Перетащил ноду куда угодно — осталась там, процесс не сменился.
- [ ] Перетащил ноду на чужой контейнер — НЕ объединилась.
- [ ] Combo «Процесс/Воркер» по-прежнему меняет процесс/воркер.
**Out of scope:** drag-reorder плагинов внутри процесса.

### Task 2 — Авто-персист позиций в рецепт
**Цель:** позиция сохраняется в рецепт автоматически при перемещении → перезапуск/cold-start её восстанавливает.
**Файлы:** `presenter.py`, `io.py` (`graph_to_blueprint`/positions), сериализация рецепта.
**Шаги:**
1. На drag-end (после Task 1) — записать позицию в `_gui_positions` и **дебаунс-сохранить** в активный рецепт (`metadata.gui_positions`) через существующий save-путь (`export_topology_with_positions` / `save_to_active_recipe`).
2. Дебаунс (напр. QTimer 300–500 мс), чтобы не писать файл на каждый пиксель.
3. Убедиться: позиции — это GUI-metadata, на runtime-топологию не влияют (editor↔runtime decoupling сохраняется).
**Acceptance:**
- [ ] Подвинул ноды → перезапустил приложение → позиции те же (без «Раскладки»).
- [ ] `metadata.gui_positions` в рецепте обновляется после перемещения.
- [ ] Runtime не перезапускается от изменения позиций.
**Out of scope:** undo/redo позиций.

### Task 3 — Персист фиксации (lock) в рецепт
**Цель:** закреплённые ноды остаются закреплёнными после перезапуска.
**Файлы:** `presenter.py`, `io.py`.
**Шаги:**
1. Сохранять `_locked_nodes` в `metadata.locked_nodes` (список node_id) тем же save-путём.
2. На загрузке (`load_topology_from_config`) восстанавливать `_locked_nodes` → применять `ItemIsMovable=False` + рамку при построении сцены.
**Acceptance:**
- [ ] Закрепил ноду → перезапуск → осталась закреплённой (не двигается, рамка есть).
- [ ] Открепил → перезапуск → свободна.
**Out of scope:** массовая фиксация/групповые операции.

### Task 4 — Чистая загрузка из позиций, «Раскладка» опциональна
**Цель:** на старте граф выглядит «как надо» из рецепта; «Раскладка» — ручной опциональный инструмент.
**Шаги:**
1. Прописать `gui_positions` в рецепт `line_filter_inspect.yaml` (и проверить дефолтные рецепты) — разово через GUI (подвинуть + сохранить) либо вручную, чтобы из коробки не было «криво».
2. Дефолтный кластер (`_node_position` fallback) — только для НОВЫХ нод без сохранённой позиции.
3. «Раскладка» (Sugiyama) остаётся по кнопке/клавише L — не трогаем, не вызываем авто.
**Acceptance:**
- [ ] Cold-start рецепта с позициями — граф сразу корректный, «Раскладку» жать не нужно.
- [ ] Новая нода (drop из палитры) — встаёт в разумное место, не накладывается.

## Верификация
- Юнит: `io.py` round-trip (gui_positions + locked_nodes), `presenter` (drag → позиция без MovePlugin; load восстанавливает lock).
- qt-mcp smoke (обязателен — [[feedback_qt_mcp_smoke_verification]]): подвинуть ноды свободно; проверить, что процесс не меняется; перезапуск → позиции/фиксация на месте; combo меняет процесс.
- `python scripts/validate.py`.

## Замечания / память
- Позиции/фиксация — **GUI-metadata рецепта**, не влияют на runtime ([[project_pipeline_editor_runtime_decoupled]]).
- Процесс-контейнер следует за **членством**, не за позицией — нода, оттащенная наружу, остаётся в своём процессе (членство меняет только combo).
- Авто-сохранение позиций — дебаунс, чтобы не дёргать файл.
