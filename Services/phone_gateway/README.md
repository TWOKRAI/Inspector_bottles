# Services.phone_gateway

Приём **фотографии** и **слова** с телефона по WiFi — без установки приложения на
телефон. Телефон открывает страницу в браузере, отправляет снимок (снять или выбрать
из галереи) и/или слово; ПК использует фото как кадр источника (вместо вебкамеры),
а слово кладёт в state store (для режима распознавания букв).

## Архитектура

Зеркалит `Services.hikvision_camera`: ядро (framework-agnostic) + тонкий source-плагин.

```
Services/phone_gateway/
  gateway.py   — PhoneGateway: HTTP-сервер (stdlib) + хранилище latest_frame/latest_word
  imaging.py   — decode_image (EXIF-поворот) + letterbox (вписать без искажений)
  netinfo.py   — local_ip(): локальный IP ПК для ссылки на телефон
  qr.py        — QR-коды (опционально, через segno; нет пакета → None)
  web.py       — HTML-страница для телефона (фото + слово, две кнопки)
  plugin/      — PhoneCameraPlugin (source, name="phone_camera")
```

Сервер крутится **внутри процесса плагина** — фото попадает в `produce()` напрямую,
без межпроцессной передачи изображений.

## HTTP-эндпоинты

| Метод | Путь      | Назначение |
|-------|-----------|------------|
| GET   | `/`       | HTML-страница для телефона |
| GET   | `/health` | `{"ok": true}` |
| POST  | `/frame`  | сырые байты картинки (JPEG/PNG) → latest_frame |
| POST  | `/word`   | текст UTF-8 → latest_word |

Картинка и слово шлются как raw body (fetch с Blob/строкой) — без multipart, поэтому
на сервере не нужен разбор форм.

## Использование

`phone_gateway` — **always-on сервис** в фундаменте топологии
([backend/topology/base.yaml](../../multiprocess_prototype/backend/topology/base.yaml)),
есть в каждом рецепте. Сервер приёма **включается/выключается** из GUI:
вкладка **Services → Телефон → «Включить» / «Выключить»** (`auto_start: false` —
по умолчанию выключен). Панель показывает адрес, QR, статус и последнее слово.

Кадр-фото — выход источника `phone_camera` (порт `frame`, SHM-слот
`camera_9_frame`); рецепт, который инспектирует фото, читает этот источник.

### Подключение телефона

1. Телефон и ПК в одной сети WiFi.
2. В Services → Телефон — адрес (`http://<IP-ПК>:8080`) и QR; нажать «Включить».
3. Открыть адрес/QR в браузере телефона → выбрать/снять фото → «Отправить».
4. ⚠️ При первом запуске разрешить Python в **брандмауэре Windows** (частные сети),
   иначе телефон получит «соединение отклонено».

### Слово (режим букв)

Отправленное слово публикуется в state store:
`processes.<process_name>.state.phone.word` (+ `word_seq`, `word_ts`).
Потребитель (режим распознавания букв) подписывается на этот путь.

## QR-код (опционально)

`qr.py` генерирует QR через **segno** (pure-python). Без пакета функции возвращают
`None`, GUI показывает ссылку текстом. Установка: `uv pip install segno`.

Один QR умеет **либо** открыть URL, **либо** подключить к WiFi — не оба сразу
(нет такого стандарта). `make_qr_svg(url)` — открыть страницу;
`make_wifi_qr_svg(ssid, password)` — подключиться к сети.
