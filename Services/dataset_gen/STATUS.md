## Производительность

~3.8 ms/кадр в 1 поток (128×128, все аугментации пресета вкл.) ≈ 260 кадров/с;
датасет 33×300=9900 кадров ~40s. С `num_workers>0` в torch — линейно быстрее.
Init движка (33 класса + детекция симметрий) ~1s.

## Что сделано

- Ядро: `GeneratorConfig` (Pydantic v2, YAML-пресеты), `SpriteCatalog`
  (RGBA-эталоны + фоны/процедурные), `compose` (rotate-expand → crop-alpha →
  alpha-blend), `augment` (фотометрия полным кадром ПОСЛЕ композиции, порядок
  сцена→оптика→сенсор→кодек: блик, тень, окклюзия/cutout, blur, motion blur,
  яркость/контраст, температура, сдвиг каналов, шум, JPEG), `symmetry`
  (авто-детектор none/180/full + кодирование угла), `DatasetEngine`
- Режимы вывода на одном движке: `export_dataset` / `export_splits`
  (train/val/test папками, свой rng на сплит) и `SyntheticDataset`
  (torch, лениво, детерминизм при num_workers через rng=(seed, idx))
- QC: `save_preview_grid` — сетка с подписями (PIL, кириллица)
- Пресет `ru_letters_disk.yaml` (33 буквы, 128×128, 0–360°, glare+motion+
  shadow+occlusion+channel_shift) + генератор эталонов `tools/make_ru_letter_sprites.py`
- Тесты: 63 passed (+1 skip torch) — contract-style, синтетические фикстуры

## Ключевые решения

- Полупиксельный центр вращения: в детекторе симметрии центр (size-1)/2,
  иначе ложные «несимметрии» (сдвиг на пиксель при 180°)
- `rel_threshold`: относительный критерий 180°-симметрии — абсолютная метрика
  «разбавляется» большой инвариантной областью (буква на диске)
- Глиф в эталоне центрируется по фактическому bbox чернил, не по метрикам
  шрифта (anchor="mm" смещает букву на ~2.5px → ломает симметрию)

## Метаданные и иерархия классов (2026-06-13)

- `meta.yaml`/`meta.json` в любой папке каталога классов — разметка узла,
  наследуется сверху вниз (родитель → подкласс), `core/metadata.py:ClassMeta`.
  Поля: display_name, symmetry (переопределение), tags + произвольные (→ extra).
- Иерархия: класс = листовая папка со спрайтами; путь `letters/А` сохраняется
  в метке (`SampleLabel.class_path`). Конфликт листовых имён → явная ошибка.
- Симметрия приоритет: глобальный override → meta.yaml → авто-детектор.
- Экспорт пишет `classes.json` — реестр классов со всей разметкой (index, name,
  path, symmetry, sprite_count, meta) — «файл разметки» датасета для обучения.
- Генератор спрайтов букв пишет meta.yaml-шаблон в каждый класс.

## Слабые места — закрыты (2026-06-13)

- Фоны из папки фото: `backgrounds_dir` сканируется рекурсивно (подпапки-
  категории belt/tray/table), фон под кадр выбирается случайно и кропается.
  Процедурные фоны (когда папка не задана) расширены до 4 типов текстур
  (`core/backgrounds.py`): градиент, шлифованный металл, лента с планками,
  пятнистая поверхность. e2e подтверждён (preview_photo_bg.png).
- Каталог классов терпит метаданные: служебные папки (`_meta/`, `.x`) и файлы
  верхнего уровня игнорируются, классами считаются только подпапки со спрайтами.
- parquet: переписан на pyarrow напрямую (pandas больше не требуется),
  тест через importorskip.
