# word_layout — раскладка слова для робота-сортировщика

Processing-плагин: по целевому слову, координатам и потоку распознаваний
(`ml_inference` → буква + угол) выдаёт роботу задание на укладку каждого диска —
**куда положить и насколько довернуть**.

Недостающее звено между распознаванием буквы+угла и укладкой (см. рецепт
`hikvision_letter_robot`, узел `layout`).

## Поток

```
ml_inference.predictions [{label, angle_deg, angle_valid, confidence}]
        │
   word_layout  ── потоковый жадный матчинг буквы в слот слова ──►
        │
   robot_job {x_mm, y_mm, z_mm, r_deg, char, slot, raw_angle_deg}  → robot_io (когда подключён)
```

Конвейер-сортировщик: диски едут по одному, плагин фиксирует распознанный диск и,
если буква есть в слове, кладёт её в первый свободный слот; дубли буквы — нужное
число раз, потом пропуск; слово готово, когда все слоты заполнены.

## Входы / выходы

| Порт | Тип | Назначение |
|------|-----|-----------|
| `predictions` (вход) | list[dict] | топ-K от ml_inference (label/angle_deg/angle_valid) |
| `word` (вход, опц.) | str/dict | целевое слово (иначе register `target_word`) |
| `trigger` (вход, опц.) | any | «взять диск» при `use_trigger=True` |
| `robot_job` (выход) | dict | поза роботу `{x_mm, y_mm, z_mm, r_deg, …}` |
| `frame` (выход) | image | pass-through |

## Раскладка (register-поля, живые)

- **`use_pitch=True`** (осн.): от ПЕРВОГО диска (`first_x`/`first_y`) по направлению
  `line_angle_deg` (90 = вдоль +Y: X постоянный, Y растёт) с шагом `pitch_mm`
  (`0` = авто = диаметр `2·disk_radius_mm`). Слово любой длины — один шаг.
- **`use_pitch=False`**: равномерно между `first` и `last` (`last_x`/`last_y`).
- Пробел в слове = `word_gap_slots` пустых ячеек (1 = пустота в один диск).
- `place_z_mm` — высота Z укладки в команде роботу.

## Угол (доворот)

`r_deg = wrap180(angle_zero_deg − sign·angle_модели)`, где `sign = −1` при `angle_invert`.
Доворот у КАЖДОГО диска свой (диски приезжают под случайным углом); финальная
ориентация общая (`angle_zero_deg`, дефолт 0 = прямо). При `angle_valid=False`
(полная симметрия буквы, напр. «О») доворот = 0. `angle_zero_deg`/`angle_invert` —
калибровка нуля и направления модель↔робот (подбирается на железе).

## Детект «нового диска»

Авто: новый диск = сменилась буква ИЛИ была пауза (нет диска). Дедуп по БУКВЕ
(угол дрожит ±1-2° — в подпись не берём). `settle_frames` — кадров стабильности
(**1** для тракта «1 кадр на диск», напр. crop по триггеру линии). `use_trigger=True`
+ wire сигнала в `trigger` — брать по внешнему сигналу.

## Команды

- `reset_word` — сбросить раскладку (начать слово заново).

## Статус / ограничения

- Раскладка и поза `{x,y,z,r}` считаются и публикуются (лог + state
  `processes.<proc>.state.word_layout`).
- **Движение робота ещё НЕ подключено**: `robot_io` не вписан, доворот `r` в Lua-протоколе
  робота (`send_job` без поворота) — отдельный шаг (см. план).
