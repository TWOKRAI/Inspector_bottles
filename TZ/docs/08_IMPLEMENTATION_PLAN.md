## 3. Риски и митигация

### 3.1. Технические риски

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| R1 | **Блики фольги** не позволяют детектировать дефекты | Средняя | Критическое | PoC на Фазе 1 с реальными образцами; подбор освещения (dome light, dark field); несколько типов подсветки |
| R2 | **Прозрачная плёнка ОПП** — низкий контраст дефектов | Средняя | Высокое | Специальное освещение (боковое, UV); отдельный алгоритм для плёнки |
| R3 | **Нейросеть не достигает** требуемой точности (95% recall) | Средняя | Критическое | Большой датасет; итеративное дообучение; fallback на классические методы; гибридный подход |
| R4 | **280 номенклатур** — не все типы удаётся инспектировать одинаково | Низкая | Среднее | Группировка по типам (плёнка/без, футляр/без); отдельные рецепты; пошаговая валидация |
| R5 | **Hikrobot SDK** — проблемы интеграции с Python/Linux | Низкая | Высокое | MVS SDK хорошо документирован; есть Python samples; fallback — GenICam через Aravis |
| R6 | **Связь с Delta PLC** — нестандартный протокол | Низкая | Среднее | Delta DVP/AS поддерживает Modbus TCP (стандарт); библиотека pymodbus проверена |
| R7 | **Производительность ПК** не хватает на 4 линии | Очень низкая | Высокое | Расчёт показывает запас 2-7×; оптимизация TensorRT; при необходимости — второй ПК |

### 3.2. Организационные риски

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| R8 | **Заказчик не предоставляет** образцы дефектов вовремя | Высокая | Критическое | Зафиксировать в контракте: образцы до начала Фазы 1; начать с синтетических данных |
| R9 | **Изменение ТЗ** в процессе разработки | Средняя | Высокое | Контрактные границы; change request процедура; модульная архитектура позволяет менять части |
| R10 | **Площадь 17.88×9.0 м** — не хватает для всех узлов | Средняя | Среднее | Компактная компоновка; обсудить на этапе КП |
| R11 | **72 часа непрерывной работы** (приёмка) — сбои ПО | Средняя | Высокое | Стресс-тестирование до выезда; watchdog + auto-restart; логирование всех ошибок |

### 3.3. Коммерческие риски

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| R12 | **Конкуренция:** готовые решения (Cognex ViDi, Keyence) | Высокая | Среднее | Наше преимущество: гибкость, цена, локальная поддержка, адаптация под заказчика |
| R13 | **Санкции:** невозможность купить NVIDIA GPU | Низкая | Высокое | Альтернативы: китайские GPU (Muxi, Iluvatar); CPU-only inference (медленнее но возможно); Jetson (импортозамещение) |
| R14 | **Hikrobot** задержки поставки | Низкая | Среднее | Daheng/Basler как альтернатива; GenICam совместимость |
