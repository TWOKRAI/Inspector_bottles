## 2. Матрица: Дефект → Метод детекции

| ID | Дефект | Метод 1 (основной) | Метод 2 (резервный) | Модель/Алгоритм | Уверенность |
|----|--------|-------------------|---------------------|-----------------|-------------|
| **F1** | Нахлёсты | YOLO detection (торец) | Контурный анализ: отклонение от окружности | YOLOv8 + cv2.findContours + cv2.minEnclosingCircle | Высокая |
| **F2** | Пятна | YOLO detection | Blob detection + цветовая аномалия | YOLOv8 + cv2.SimpleBlobDetector | Высокая |
| **F3** | Загрязнения | YOLO detection | Текстурный анализ (LBP/Gabor) | YOLOv8 + skimage.feature.local_binary_pattern | Средняя |
| **F4** | Мех. повреждения | YOLO detection | Edge anomaly detection | YOLOv8 + Canny + морфология | Средняя |
| **F5** | Замятия | YOLO detection | 3D-профиль (если доступен) | YOLOv8 + gradient magnitude analysis | Средняя |
| **P1** | Нет плёнки | Классификация + яркость | Разница отражения (фольга vs плёнка) | ResNet classifier / порог яркости | Высокая |
| **P2** | Повреждение плёнки | YOLO detection | Текстурная аномалия | YOLOv8 + Gabor filters | Средняя |
| **P3** | Морщины плёнки | YOLO detection | Gradient direction analysis | YOLOv8 + Sobel + direction histogram | Средняя |
| **P4** | Нет даты | OCR → проверка наличия | Classification (дата есть/нет) | PaddleOCR / Tesseract + regex | Средняя |
| **P5** | Загрязнения плёнки | YOLO detection | Цветовая сегментация | YOLOv8 + HSV threshold | Средняя |
| **P6** | Смещение плёнки | Контурный анализ | YOLO detection | cv2.matchTemplate + edge alignment | Высокая |
| **B1** | Повреждение футляра | YOLO detection | Edge/contour analysis | YOLOv8 | Средняя |
| **B2** | Нет склейки | YOLO detection | Геометрический анализ клапанов | YOLOv8 + template matching | Средняя |
| **B3** | Загрязнение футляра | YOLO detection | Blob detection | YOLOv8 | Высокая |
| **B4** | Нет даты футляра | OCR → проверка | Classification | PaddleOCR / Tesseract | Средняя |

## 3. Приоритеты реализации

### Фаза 1 — MVP (минимальный набор для запуска)

| Приоритет | Дефекты | Обоснование |
|-----------|---------|-------------|
| P0 | P1 (нет плёнки) | Простая классификация, критичный дефект |
| P0 | F4, F5 (мех. повреждения, замятия) | Самые частые дефекты |
| P0 | F2 (пятна) | Хорошо детектируются нейросетью |
| P1 | F1 (нахлёсты) | Видны на торце, хорошо формализуемы |
| P1 | P4, B4 (нет даты) | OCR — отработанная технология |

### Фаза 2 — Полный набор

| Приоритет | Дефекты | Обоснование |
|-----------|---------|-------------|
| P2 | P2, P3 (повреждение/морщины плёнки) | Требуют хорошего освещения и датасета |
| P2 | F3, P5 (загрязнения) | Граница между "допустимо" и "брак" размыта |
| P2 | B1, B2, B3 (футляр) | Не все номенклатуры имеют футляр |
| P3 | P6 (смещение плёнки) | Требует точной калибровки |

## 4. Требования к датасету

### 4.1. Сбор данных

| Этап | Действие | Количество | Кто |
|------|----------|------------|-----|
| 1 | Получить образцы дефектных рулонов от заказчика | 50-100 шт (разные дефекты) | Заказчик |
| 2 | Снять на макете (камера + освещение) | 500-1000 фото на класс | Мы |
| 3 | Разметить (bounding boxes) | 5000-10000 аннотаций | Мы / аутсорс |
| 4 | Аугментация (rotate, flip, brightness, noise) | ×5-10 от исходного | Автоматически |
| 5 | Валидация на реальном потоке | Непрерывно при пуско-наладке | Мы + заказчик |

### 4.2. Минимальный датасет для MVP

- **200-500 изображений на класс дефекта** (после аугментации ~2000-5000)
- **1000+ годных образцов** (для negative mining)
- Формат: COCO JSON или YOLO txt
- Инструмент разметки: CVAT, Roboflow, или LabelStudio

### 4.3. Образцы от заказчика (из ТЗ)

> "Образцы дефектных рулончиков будут предоставлены для Поставщика оборудования"

Это ключевой пункт. Без реальных образцов невозможно обучить модель. Нужно в КП зафиксировать:
- Заказчик предоставляет образцы дефектов до начала разработки ПО
- Количество: минимум 10 шт каждого типа дефекта
- Формат: физические рулоны для съёмки на нашем оборудовании

## 5. Метрики качества

| Метрика | Целевое значение | Примечание |
|---------|-----------------|------------|
| Recall (чувствительность) | ≥ 95% | Пропуск дефекта — критичен |
| Precision (точность) | ≥ 90% | Ложные срабатывания → потери производительности |
| False Positive Rate | ≤ 10% | Годные рулоны отмечены как брак |
| False Negative Rate | ≤ 5% | Дефектные рулоны пропущены |
| Время инспекции | ≤ 500 мс | На 1 рулон (все камеры) |
| Доступность системы | ≥ 99.5% | 24/7 режим |

## 6. Особенности по типам продукции

| Тип | Особенности инспекции |
|-----|----------------------|
| В плёнке ОПП | Полный набор проверок (фольга + плёнка + дата) |
| Без плёнки | Только дефекты фольги; прямой контакт — блики сильнее |
| С футляром | Дополнительно: проверка футляра (склейка, повреждения, дата) |
| Без футляра | Стандартный набор без проверок B1-B4 |

Рецепт для каждой номенклатуры должен указывать, какие проверки активны.
