## Что доказано

1. **Шторм воспроизведён живьём** и совпал с диагнозом плана: `state.changed` идёт в
   never-drop system-очередь gui, та забивается в потолок (100), **команды физически
   не могут войти** — отсюда 85 % отказов отправки и слепой gui.
2. **Флип гасит шторм полностью.** `system_evict_blocked` и `errors` — строго 0,
   доставка 100 %, очереди gui пусты, gui отвечает на команды.
3. **Механика 1.2 работает как задумано.** `sent_via_targets.state=247` — конверты
   ушли в `{proc}_state`; `queue_data_evicted=89` — drop_oldest роняет устаревшие
   снимки (клиент восстановит resync'ом) **вместо** блокировки control-plane.
   `system_evict_blocked=0` подтверждает, что state-класс не never-drop.
4. **OFF-путь честен.** То, что в baseline забивается именно `system`, а `.state`-
   счётчика у PM нет, доказывает: при OFF поведение прежнее, бит-в-бит.

## Task 1.4 — ui_*-плоскость доказана парой (тот же день, флаги ON, `QT_MCP_PROBE=1`)

Закрывает вечное «ui_* NA» прошлых верификаций.

| Шаг | Результат |
|---|---|
| `ui_tap` | `success`, `subscriber=backend_ctl.0b245cfb5956`, `command=ui.event`, `sources=[gesture, command]` |
| `ui_tap_ping` | `events_sent=3`, `send_errors=0` → в `events_page(plane=ui)` пришло `kind=ping`, note совпал |
| **Реальный клик** (qt-mcp, кнопка «Audit log») | `ui.event`: `kind=button`, `text="Audit log"`, `widget=QPushButton`, полный `path` в дереве виджетов, `_address=[backend_ctl, 0b245cfb5956]`, `dropped=0` |
| `ui_untap` → повторный клик | **тишина**: 0 событий по курсору |

Плоскость работает end-to-end: подписка → доставка жеста с атрибуцией → отписка глушит.

### Побочная находка: «No handler for key» врёт про причину

Первый `ui_tap` сразу после boot упал с `No handler for key 'ui.tap.subscribe'` — при том,
что команда **есть** (`introspect_handlers` показал её в `commands` gui). Причина: gui был
объявлен готовым по liveness-fallback (`'gui' ready via liveness-fallback (event не пришёл
за 5.0s)`) ещё ДО регистрации своих команд; вызов в это окно получает сообщение, которое
читается как «фичи не существует», хотя правда — «процесс ещё не готов». На прогретой
системе тот же вызов проходит без изменений в коде.

Гипотеза «сломана обёртка драйвера» проверена и **опровергнута** — `ui_tap` корректен.
Residual для Фазы 4/5: различать «нет такого хендлера» и «хендлеры ещё не подняты».
