# docs/audits/ — Аудиты кодовой базы

Снимки состояния кода (проблемы + backlog улучшений), сделанные многоагентным аудитом.
Каждый аудит — справочник со стабильными ID находок (`H1`, `M-leak-1`, …), на которые можно ссылаться.

> **Для агентов:** перед действием по находке перечитай актуальный код по `file:line` — он мог измениться с даты аудита.

## Соглашения

- **Severity:** HIGH (ломает работу/данные) → MEDIUM (заметный риск/smell) → LOW/INFO (мелочь). Critical отдельно если есть.
- **Верификация:** находки critical/high проходят состязательную проверку (скептик пытается опровергнуть). Отклонённые попадают в раздел «Ложные срабатывания» — туда же заглядывай, прежде чем заводить «новый баг».
- **ID-схема:** `H#` (high), `M-<тема>-#` (medium по темам), `L-<тема>` (low). Темы: leak, err, race, perf, dom, cfg, lay, dead, arch, god.
