Правило владельца (2026-07-13, на ревью G.2): «если что-то не используется сейчас — не значит что не нужно, конструктор-фреймворк универсален».

**Why:** по этим контрактам (ISQLManager.execute_command, формы конвертов, Message.set_command) будет собрано следующее приложение из «рыбы»; сломанный неиспользуемый путь = баг, найденный через полгода в чужом приложении.

**How to apply:** в ревью фреймворка находки на путях «без живых вызывающих» НЕ понижать до заметок — публичный API либо чинится, либо отклоняет вход громко и понятно (не тихой валидацией), либо честно помечается legacy/frozen в докстринге (freeze-не-kill, см. [[freeze-over-kill]]). Зелёные тесты проверяют используемое; ревью проверяет обещанное. См. [[test-params-hide-defect-window]], [[review-economy-tiers]].
