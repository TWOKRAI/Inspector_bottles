`test_fencing_live.py::test_stale_dropped_after_instance_replace_green` красный **не из-за продукта**. Живой лог из точки решения (`restart_process`) показывает исправную цепочку: `reuse_cfg=False`, очереди пересозданы, `bumped=True`. Не долетает сам стейл: `restart_process` добивает старый инстанс с подтверждением смерти ДО того, как поднять incarnation, поэтому к моменту бампа во входящей очереди PM обычно уже нечего дропать.

Замер 2026-07-21: тест 3/3 красный, проба 1 из 3 зелёная (случайный `fence_dropped=1`). Тот единственный зелёный чуть не увёл диагностику в вывод «механизм работает».

**Why:** тест утверждает то, что система выдаёт лишь по везению — это флейк по конструкции, а не регресс.

**How to apply:** детерминировать стейл нельзя инъекцией через драйвер ([[project_backend_ctl_socket_bypasses_mw]]) — штамп ставит send-мидлварь отправителя, а сокет минует receive-мидлварь. Решение о форме теста не принято; ghost-гонку соседнего теста закрыл гейт топологии (ADR-SS-019), fencing тут ни при чём — он про ЗАМЕНУ инстанса, а снятый switch'ем процесс удалён, его incarnation никто не бампит.
