Живой прогон `webcam_sketch` с `BACKEND_CTL=1` (2026-07-21) — первая проверка инструмента
по назначению после Фазы 1 плана `backend-ctl-proof-discipline`. Дал три находки, ни одна
из которых не ловилась ни одним из 504 unit- и 36 live-тестов.

**1. Ротация логов молча не работает.** Настроено `max_size=10MB`, `backup_count=5`
(`logger_module/configs/logger_manager_config.py:31-32`) → потолок ~60 МБ. Фактически
`logs/ProcessManager/messages.log` = **645 МБ**, весь `logs/` = 1.5 ГБ (из них
`logs/prototype_2/` — 779 МБ мёртвого веса от удалённого в 2026-06 прототипа).
Причина — `logger_module/channels/log_channel.py:51-56`: `except PermissionError: pass`.
На Windows 36 процессов конкурируют за файл, ротация падает, ошибка глотается без
счётчика и без предупреждения. Усугубляет `Command '<x>' executed successfully in 0.000s`
на уровне **INFO**, тысячи строк в секунду.

**2. ~23% отправок ProcessManager падают, причина невидима.** Замер: за 20 с БЕЗ
подключённого драйвера `sent_ok +1144`, `errors +345` — собственное состояние системы,
не артефакт подписок (проверено экспериментом с отключением). Причина не видна принципиально:
`router_module/core/router_manager.py:365` считает ошибку и пишет причину через `_log_debug`.
Это ошибки ОТПРАВКИ, не приёма (первичный диагноз «99% принятого падает» был неверным).

**3. `config_reload` врёт про `log_level`.** Ответ `{'success': True, 'applied':
{'log_level': 'DEBUG'}}` — и ни одной DEBUG-строки в логах. Ложный успех в самом
инструменте доказательности.

**Побочно:** MCP-сервер `backend_ctl.mcp_server_sdk` держит код на момент своего старта —
`telemetry_snapshot` через MCP не показывал поля `ingest_active`/`ingested_total`, добавленные
коммитом `c9e14cef`, тогда как прямой вызов из свежего процесса показывал. Версия сборки
агенту не сообщается. Перед проверкой свежих правок через MCP — перезапускать сервер или
идти напрямую через `BackendDriver`.

**Не доказано:** транзиентная задержка 3-4 с в тракте (наблюдал владелец, ушла сама).
Замер шёл уже после восстановления: очереди по нулям, латентность по стадиям 62/19/25/1.6 мс.
Истории нет — `telemetry_history` копит только с момента подписки. Кандидат-гипотеза:
конкуренция за GPU с Ollama (TEED на 4 ГБ RTX 3050 — задокументированное узкое место рецепта).
Проверять только с включённым `record_start` в момент следующего эпизода — это ровно тот
реальный случай, которого не хватало recorder'у для оправдания (Task 4.2 плана).
