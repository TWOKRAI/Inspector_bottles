NODE_MOVE (GUI-only) и ROLE_UPDATE (auth-домен) остаются на отдельной шине. **Целевая архитектура:** unidirectional/CQRS — презентер→`dispatch(DomainCommand)`→apply→save(store-publishes TopologyReplaced)→publish events; view обновляется ИЗ событий; PipelineModel→производная domain (refresh на TopologyReplaced). **G.4.1 (построено):** `ProjectHistory` (adapters/dispatch/history.py — snapshot-стек before/after Project + coalescing по coalesce_key + max_history=50; чистый, без Qt/IPC/holder); orchestrator получил `undo/redo/can_undo/can_redo/history/clear_history` + `dispatch(coalesce_key, undoable)`; `_restore`=topology_repo.save+holder.set (store публикует TopologyReplaced→презентеры reload; granular-события при undo НЕ переигрываются — семантика legacy ActionBus сохранена); domain `CommandDispatcher` Protocol расширен + sidecar `HistoryEntry`; FakeCommandDispatcher/_FakeDispatcher/_FakeDispatcherWithBus обновлены.
