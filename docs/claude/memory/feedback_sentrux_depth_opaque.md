sentrux `depth` (root_causes.depth) **не соответствует** ни каталожной вложенности пакетов, ни длине цепочки импортов — обе гипотезы опровергнуты замером (2026-07-10):

- Проба: физически убрал самый глубокий по каталогам пакет (`services/robot/calibration`, 7 уровней) → `depth raw` остался **6**, score 0.5714 не сдвинулся.
- Скрипт по графу импортов: длиннейшая цепочка = **17 пакетов / 21 файл** ≠ raw 6.

Значит метрика нормализована/агрегирована, а её root-cause закрыт за **sentrux Pro**. Снаружи точное определение не восстанавливается.

**Why:** задача 5.18 (depth ≥ 0.60) была сформулирована как «сплющить каталоги god-split», но перенос папок метрику не двигает — впустую перелопатил бы 12+9-файловые пакеты с десятками импортёров на core-структуре ради интерим-порога.

**How to apply:** не гоняться за точным числом `depth` вслепую. Если нужно поднять — либо получить sentrux Pro root-cause, либо перебор (правка→rescan→смотреть число), либо переформулировать под РЕАЛЬНУЮ слоистость (самые длинные внутренние цепочки — `logger_module` core→logger_core→channels→interfaces→log_config→configs, 6 hop). Здоровые ориентиры: каталожная вложенность ≤4–5, внутренняя слоистость модуля ≤4–5 hop. Связано с [[feedback_fewer_layers]].
