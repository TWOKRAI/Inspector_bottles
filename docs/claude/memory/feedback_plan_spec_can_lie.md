Спека задачи в плане — гипотеза, а не истина. Task 2.1 (`truth-holes-closure`) требовал поле
`manual_restarts`; ревью Fable показало, что имя лживо: авто-рестарт монитора приходит в PM
**той же командой** `process.restart`, поэтому счётчик ловит и его. Реализовано как
`instance_restarts` — «любая замена инстанса оркестратором», без заявления о происхождении.

**Why:** имя поля — часть контракта наблюдаемости. Ложное имя в инструменте «показывающем правду»
хуже отсутствия поля: оператор делает уверенный неверный вывод («кто-то рестартовал руками»
вместо «процесс упал»). Это тот же класс, что [[feedback_swallowed_failure_class]] —
следствие без причины.

**How to apply:** прежде чем реализовать имя/семантику из спеки — проверить по коду, кто ещё
идёт этим путём. Если путей больше одного, имя не должно называть один из них. Расхождение с
планом фиксировать прямо в плане (строка «правка спеки по ревью») и в трейлере `Rejected:`.

См. также [[feedback_plausible_is_not_verified]], [[project_telemetry_coherence_remediation]].
