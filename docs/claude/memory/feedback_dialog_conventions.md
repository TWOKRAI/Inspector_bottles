Владелец (2026-07-13, RS-4) дважды вмешался в диалог несохранённых правок: (1) потребовал кнопку «Сохранить» (а не только «потерять/отмена»), (2) отверг формулировку «Продолжить без сохранения» в пользу стандартной «Не сохранять».

**Why:** владелец ожидает конвенции macOS/редакторов (Save / Don't Save / Cancel) — самодельные формулировки заставляют задумываться; «Сохранить» обязана быть кнопкой по умолчанию (безопасный исход — правки не теряются случайно).

**How to apply:** любые новые подтверждающие диалоги в GUI — сначала стандартная конвенция платформы, не изобретать текст. Общий helper уже есть: `frontend/widgets/dialogs/unsaved_changes.py::confirm_unsaved_changes` — новые точки «опасное действие при dirty» обязаны использовать его, не городить свой QMessageBox. См. [[formal-review-before-merge]].
