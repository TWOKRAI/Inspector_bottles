Решение владельца (2026-07-13, после ревью G.6): полный 8-угольный протокол ревью на каждую задачу жжёт токены. Схема:

| Уровень | Когда |
|---|---|
| Полное 8-угловое (финдеры+верификаторы Sonnet, свод Fable) | рисковые вскрытия: hot-path, конкурентность, контракты доставки (Ф7 G.2/G.3/G.4) |
| Лёгкое: reviewer-агент (Opus), 2–3 угла (line-by-line + cross-file + фасады) | S-задачи, механика, пост-ревью-фиксы |
| Группой: одно ревью на пару смежных S-задач одной ветки перед одним merge | напр. G.6-фиксы + G.1 |

**Why:** гейт «формальное ревью до merge» сохраняется всегда (он окупается — G.6 дал 3 CONFIRMED на «простой» задаче), меняется только глубина/частота; главная экономия — финдеры/верификаторы НЕ на Fable + делегировать весь протокол reviewer-агенту, чтобы Fable читал только итог.

**How to apply:** перед ревью выбрать уровень по риску задачи (конкурентность/контракты → полный), для пары смежных S-задач копить одну ветку и ревьюить разом. См. [[model-split-impl-vs-review]], [[formal-review-before-merge]].
