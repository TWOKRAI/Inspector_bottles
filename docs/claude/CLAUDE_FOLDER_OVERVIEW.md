# Обзор папки `.claude/` (Claude Code)

Краткий справочник: что лежит в конфигурации Claude Code, зачем это нужно, как устроены настройки и хуки. **Не подмешивается в автоконтекст** — читать по запросу. Оглавление по файлам: `.claude/README.md`.

---

## Назначение

Папка `.claude/` — это **проектная конфигурация Claude Code** (не Cursor): правила репозитория, разрешения на команды, MCP, субагенты, скиллы, slash-команды и shell-хуки. Отдельно от документации фреймворка (`Inspector_prototype/multiprocess_framework/docs/`).

---

## Структура (превью)

| Путь | Роль |
|------|------|
| `CLAUDE.md` | Главный проектный контекст: архитектура, стек, правила, MCP qex |
| `CLAUDE-SETUP.md` | Как развернуть конфиг на другой машине / в другой клон |
| `README.md` | Индекс: таблицы агентов, скиллов, команд, хуков |
| `settings.json` | Разрешения Bash, хуки Pre/Post, язык, status line |
| `mcp.json` | MCP **qex** (Qdrant + Ollama), путь к `qex-mcp-v2`, переменные окружения |
| `agents/*.md` | Субагенты (узкие роли: архитектор, IPC, тесты, PyQt, security) |
| `skills/*/SKILL.md` | Скиллы — процедуры по явному вызову (`/skill-name`) |
| `commands/*.md` | Slash-команды — готовые промпты и команды shell |
| `hooks/*.sh` | Скрипты до/после инструментов (безопасность bash, ruff) |

Тяжёлая документация вынесена в `docs/claude/`, чтобы **не раздувать постоянный контекст** чата.

---

## Что загружается когда

- **`CLAUDE.md`** (и при наличии `CLAUDE.local.md`) — ориентируйтесь как на «базовый системный промпт репозитория».
- **Агенты, скиллы, slash-команды** — только при явном использовании; **не** увеличивают каждый диалог.
- **`settings.json` / `mcp.json`** — конфиг среды, в текст модели сами по себе не «вшиты».

---

## `settings.json`: не MCP, а политика и автоматизация

### Permissions (`allow` / `ask` / `deny`)

- **allow** — типичные безопасные команды (`pytest`, `ruff`, `python`, локальный `curl` к localhost, часть `docker` для статуса/логов/старта).
- **ask** — потенциально разрушительное или необратимое (`rm`, произвольный `curl`, `docker rm/stop/kill`, `docker system`).
- **deny** — жёсткий запрет (`sudo`, опасный `git push -f`, `git reset --hard`, агрессивный `docker system prune` и т.д.).

Эффект: меньше случайных повреждений и меньше лишних запросов подтверждения на рутину.

### Hooks

- **PreToolUse + Bash** → `hooks/validate-safe-command.sh` — при срабатывании паттерна опасной команды выход с кодом **2** блокирует выполнение.
- **PostToolUse + Edit|Write** → `hooks/autoformat-python.sh` — для изменённых `.py`: `ruff format` и `ruff check --fix` (если `ruff` найден; иначе тихий выход).

### Прочее

- **`preferredLanguage: ru`** — предпочтение русского в ответах.
- **`statusLine`** — одна строка в UI: текущая ветка git и индикатор, запущен ли контейнер `qdrant` (для стека семантического поиска).

Замечание по ОС: хуки и `statusLine` написаны под **bash** (`jq`, `grep`, `printf`). На Windows без Git Bash/WSL поведение может отличаться — при «пустом» статусе или неработающих хуках проверить среду выполнения Claude Code.

## MCP (`mcp.json`)

Подключается сервер **qex**: семантический поиск кода (`search_code`, `index_codebase`, `get_indexing_status`) при работающих **Qdrant** и **Ollama**. Путь к бинарнику и `WORKSPACE_PATH` **машинозависимы** — при переносе репозитория пути нужно обновить (или держать шаблон/локальный override в документации qex).

В репозитории параллельно может быть настроен **другой** семантический поиск (например, Milvus + claude-context в Cursor). Это разные каналы; иметь **один основной** сценарий для команды снижает путаницу в инструкциях.

## Эффективность по слоям

| Механизм | Эффект |
|----------|--------|
| `CLAUDE.md` | Меньше ошибок по структуре репо и правилам (Dict at Boundary, тесты, пути). Цена — размер контекста. |
| permissions | Баланс удобства и безопасности для Bash. |
| PreToolUse hook | Страховка от класса опасных однострочников агента. |
| PostToolUse ruff | Единообразный стиль правок без ручного прогона. |
| MCP qex | Сильная помощь на больших рефакторингах и поиске «по смыслу» при актуальном индексе. |
| Агенты / скиллы | Качество на узких задачах без раздувания каждого чата. |
| Slash-команды | Повторяемые проверки (`/validate`, `/fw-test`, холодный старт и т.д.). |

---

## Зачем много отдельных файлов

1. **Бюджет контекста** — не платить за килобайты текста в каждом сообщении.
2. **Разные режимы** — ревью архитектуры и точечный фикс требуют разного фокуса (субагенты).
3. **Онбординг** — `README.md` + `CLAUDE-SETUP.md` дают карту без чтения всех агентов.
4. **Ограждения** — permissions и хуки формализуют безопасность, а не только «просим модель быть осторожной».

## Возможные улучшения (чек-лист)

- Явно описать в одном месте: **Claude Code + qex** vs **Cursor + другой MCP** (если оба используются).
- Шаблон `mcp.json` с плейсхолдерами путей или документ «после клона заменить X».
- Убедиться, что на вашей Windows-среде реально выполняются `.sh` хуки; при необходимости дублировать логику для среды, которую использует Claude Code.
- Периодически **сжимать** `CLAUDE.md`: переносить уже усвоенное в `docs/claude/`, оставляя только то, что реально снижает ошибки (см. советы в `CLAUDE-SETUP.md`).
