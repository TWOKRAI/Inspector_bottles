## P1 — Архитектурный долг (восстановление метрик)

### Task X.3 — Категоризация плагинов в `Plugins/` (восстановление modularity)

**Проблема:** sentrux modularity упала с 0.448 → 0.242, потому что 19 плагинов теперь видятся как top-level модули. ADR-120 обещает «восстановление при дальнейших улучшениях», но плана не было.

**Подход:**
1. **Дизайн-сессия (30 мин):** определить категории. Кандидатная разбивка:

   | Категория | Плагины |
   |---|---|
   | `Plugins/sources/` | `camera_service`, `capture`, `frame_counter`, `heartbeat` |
   | `Plugins/processing/` | `grayscale`, `negative`, `flip`, `color_mask`, `blob_detector`, `region_split`, `resize`, `stitcher` |
   | `Plugins/io/` | `database`, `frame_saver` |
   | `Plugins/render/` | `render_overlay`, `renderer_compositor` |
   | `Plugins/control/` | `robot_control` |
   | `Plugins/runtime/` | `chain_executor`, `worker_pool` |

2. `git mv` 19 плагинов в категории.
3. Sed-замена `Plugins.X` → `Plugins.<category>.X` во всех:
   - Python-импортах (~25 ссылок)
   - YAML topology (~30 строк, `plugin_class:`)
   - Config string-literals (~20 ссылок в `plugin_class` configs внутри самих плагинов)
4. `PluginRegistry.discover` — `rglob("plugin.py")` уже рекурсивен, не требует изменений.
5. Обновить ADR-120: добавить раздел «Структура категорий» или создать ADR-123 «Категоризация плагинов».

**Подводный камень:** Это **второй** массовый sed по плагинам за неделю — git history усложнится. Митигейшен: один атомарный коммит «restructure: Plugins/ categorization», ясный commit message.

**Acceptance:**
- [ ] sentrux modularity ≥ 0.45 (восстановление до уровня до Phase 5)
- [ ] sentrux quality_signal ≥ 6300 (восстановление)
- [ ] Все 9 правил pass; все тесты green
- [ ] `/run-proto` запускается; PluginRegistry находит 19 плагинов

**Эстимейт:** 3-5 часов.

**Альтернатива:** оставить как есть, а в `.sentrux/rules.toml` снизить `min_modularity` до 0.20 как «новой нормы». **Не рекомендуется** — это лечение симптома, метрика теряет смысл.
