## Распределение ресурсов по уровням

### Уровень 1 — Центральный мозг (Orin NX 16GB)

**Задачи:**
- **LLM (13-20B параметров)** — диалог, reasoning, принятие решений
  - Формат: GGUF Q4/Q5 через llama.cpp или TensorRT-LLM
  - RAM: 8-12 GB
- **Голос (STT + TTS)**
  - Whisper-large-v3 (~1.5 GB) — распознавание речи
  - Piper / VITS TTS (~0.5 GB) — синтез речи
- **Планировщик поведения** — state machine / behavior tree
  - Принимает данные от всех уровней, формирует высокоуровневые команды
- **Сенсорный фьюжн** — объединение данных от камер, лидара, depth

**Почему Orin NX:** 100 TOPS + 16 GB позволяют запустить LLM 13B и Whisper одновременно без swap.

### Уровень 2 — Сенсорный хаб (Jetson Orin Nano 8GB, STEMBLOCK)

**Задачи:**
- **SLAM + локализация** — построение карты из лидара
  - Алгоритмы: Hector SLAM / Cartographer / RTAB-Map
  - RAM: 2-3 GB
- **Depth processing** — depth камера (TBD)
  - 3D point cloud → obstacle map
  - Visual odometry (с IMU)
  - RAM: 2 GB
- **Навигация** — path planning (A*, D*, DWA)
  - Локальный и глобальный планировщик
  - RAM: 1 GB
- **Резервный голос** — Whisper-small для аварийных команд если Orin NX перегружен

**Почему Orin Nano:** 40 TOPS Ampere + CUDA 12 + JetPack 6 — та же экосистема что Orin NX.
SLAM и навигация изолированы от LLM, стабильный real-time без конкуренции за GPU.
В отличие от старого Jetson Nano — полная совместимость софта с Orin NX.

### Уровень 3 — Зрительная кора (RPi 5 + AI HAT+)

**Задачи:**
- **Детекция объектов** — YOLOv8n/YOLOv9t на Hailo-8L NPU
  - 26 TOPS достаточно для 30+ FPS на 640×640
  - CPU RPi свободен — NPU работает автономно
- **Распознавание лиц** — ArcFace / InsightFace на NPU
- **Распознавание жестов** — MediaPipe или custom модель
- **ROS2 брокер** — маршрутизация сообщений между уровнями
- **Резервное управление** — если Уровень 1 или 2 упали, базовое управление

**Почему RPi:** Hailo NPU разгружает CPU полностью. RPi — отличный ROS2 node.

### Уровень 4 — Исполнительный (Arduino Mega 2560)

**Задачи:**
- **ШИМ для моторов** — 4 канала DC моторов через H-bridge
- **Управление сервами** — 8-12 каналов (манипулятор, поворот камеры)
- **Опрос энкодеров** — прерывания для точного подсчёта
- **Аварийный стоп** — hardware watchdog, кнопка E-stop
- **Базовые сенсоры** — бамперы, ИК-датчики, ультразвук (если нужно)

**Почему Arduino:** жёсткий real-time (микросекунды), нет ОС, нет задержек GC/kernel.

## Принципы разделения

| Принцип | Описание |
|---------|----------|
| **Изоляция задержек** | LLM (секунды) не блокирует навигацию (миллисекунды) |
| **Отказоустойчивость** | Падение Orin NX не убивает навигацию и safety stop |
| **Оптимальная загрузка** | Каждый процессор работает близко к своему максимуму |
| **Масштабируемость** | Можно заменить один уровень, не трогая остальные |
| **Power budget** | Можно отключать Orin NX для экономии батареи |
