# Анализ выбора MCU — Уровень 4 (Real-Time исполнение)

> **Ключевое требование:** много сервоприводов для движения, гибкость, удобство, масштабируемость.

---

## Сначала — как вообще управляют сервами в робототехнике

Есть **3 принципиально разных подхода**, и от выбора подхода зависит выбор MCU:

### Подход A: PWM-сервы + PCA9685 (классика)

```
MCU (ESP32/STM32/Arduino)
  │ I2C (2 провода)
  ├── PCA9685 #1 → 16 PWM-серв (адрес 0x40)
  ├── PCA9685 #2 → 16 PWM-серв (адрес 0x41)
  └── ...до 62 плат → до 992 серв
```

- **PWM-серва** (SG90, MG996R, DS3218) — самые дешёвые и распространённые
- Каждая серва — отдельный провод от PCA9685
- **Нет обратной связи** — не знаешь текущий угол, температуру, нагрузку
- PCA9685 генерирует PWM аппаратно, MCU свободен
- **Цена:** серва ~$3-15, PCA9685 плата ~$2-3

### Подход B: Serial Bus сервы (профессиональный)

```
MCU (STM32/ESP32)
  │ UART (1 провод данных)
  ├── Серва #1 ─── Серва #2 ─── Серва #3 ─── ... (шлейф)
  └── Все на одном проводе, каждая имеет свой ID
```

- **Bus-серва** (Hiwonder LX-16A, Feetech STS3215, Dynamixel AX-12A) — подключены шлейфом (daisy chain)
- Один UART — управляет всеми, каждая серва имеет уникальный ID (до 253)
- **Есть обратная связь** — позиция, температура, напряжение, нагрузка в реальном времени
- **Цена:** серва ~$10-30 (LX-16A ~$12, Dynamixel ~$30-90)

### Подход C: Готовая ROS Expansion Board (всё-в-одном)

```
Jetson / RPi
  │ USB/UART
  └── ROS Expansion Board (STM32 на борту)
        ├── 4× encoder моторы (шасси)
        ├── 4× PWM сервы
        ├── Serial bus сервы (шлейф)
        ├── IMU 9-axis
        └── CAN / SBUS
```

- **Готовое решение** — YAHBOOM ROS Driver Board V3.0 или Hiwonder ROS Board
- STM32 уже на плате, прошивка готова, ROS2 драйверы есть
- Подключается напрямую к Jetson через USB
- **Цена:** плата ~$40-60

## Сравнение MCU-кандидатов

### ESP32-S3

| Параметр | Значение |
|----------|---------|
| **Ядро** | Dual-core Xtensa LX7, 240 MHz |
| **RAM** | 512 KB SRAM + до 8 MB PSRAM |
| **Flash** | до 16 MB |
| **PWM** | LEDC: 8 каналов (теоретически 16, но S3 имеет баги с >2 сервами напрямую!) |
| **MCPWM** | 2 модуля × 3 пары = аппаратный motor control для DC/BLDC моторов |
| **UART** | 3 аппаратных (1 свободен для bus-серв) |
| **I2C** | 2 шины (можно 2× PCA9685 каскада) |
| **SPI** | 3 |
| **Wi-Fi** | 802.11 b/g/n |
| **BLE** | 5.0 |
| **micro-ROS** | Нативная поддержка (Wi-Fi или UART транспорт) |
| **Цена** | ~$5-7 (DevKitC-1) |

**Сервы напрямую:** ПРОБЛЕМА. ESP32-S3 имеет [задокументированные баги](https://forum.arduino.cc/t/esp32-s3-not-working-with-2-servos/1157409) с управлением >2 сервами через LEDC. Это не решается софтом — это ограничение периферии S3.

**Сервы через PCA9685:** ОТЛИЧНО. I2C → PCA9685 → 16/32/... серв. MCU не нагружен, PWM генерируется аппаратно на PCA9685.

**Bus-сервы:** ХОРОШО. 1-2 UART свободны → подключение LX-16A / Feetech шлейфом.

**DC моторы:** ОТЛИЧНО. MCPWM — лучший аппаратный motor control среди всех MCU в этом ценовом диапазоне.

**Итого:** Отличный MCU за $5, но ТОЛЬКО с PCA9685 или bus-сервами. Напрямую много серв не потянет.

---

### STM32F407VET6 / STM32F446RE

| Параметр | F407VET6 | F446RE |
|----------|---------|--------|
| **Ядро** | ARM Cortex-M4F, 168 MHz | ARM Cortex-M4F, 180 MHz |
| **RAM** | 192 KB | 128 KB |
| **Flash** | 512 KB | 512 KB |
| **PWM (Advanced Timer)** | TIM1 + TIM8 = до 14 комплементарных каналов | аналогично |
| **PWM (General Timer)** | TIM2-TIM5, TIM9-TIM14 = 20+ каналов | аналогично |
| **UART** | 6 | 6 (4 USART + 2 UART) |
| **I2C** | 3 | 3 |
| **SPI** | 3 | 4 |
| **Wi-Fi** | Нет | Нет |
| **micro-ROS** | Нативная поддержка (UART/USB транспорт) | Нативная поддержка |
| **Цена** | ~$8-15 (Nucleo / Blue Pill Plus) | ~$12-18 (Nucleo-64) |

**Сервы напрямую:** ЛУЧШИЙ В КЛАССЕ. 20+ аппаратных PWM каналов с advanced таймерами. Каждый канал — прецизионный, нет багов как у ESP32-S3. Можно управлять 16-20 сервами напрямую без PCA9685.

**Bus-сервы:** ОТЛИЧНО. 6 UART → можно подключить несколько шлейфов bus-серв параллельно.

**DC моторы:** ОТЛИЧНО. Advanced Timer с комплементарными выходами + dead-time insertion — промышленный стандарт для FOC (Field Oriented Control) BLDC моторов.

**Минусы:** Нет Wi-Fi (нужен USB/UART к Jetson). Сложнее в освоении (HAL/CubeMX). Нет OTA обновлений.

**Итого:** Промышленный стандарт для робототехники. Лучший для прямого управления сервами. Сложнее ESP32 в начале.

---

### Teensy 4.1

| Параметр | Значение |
|----------|---------|
| **Ядро** | ARM Cortex-M7, 600 MHz (NXP iMXRT1062) |
| **RAM** | 1 MB + 16 MB PSRAM (опция) |
| **Flash** | 8 MB + microSD |
| **PWM** | FlexPWM: 32 серва через IntervalTimer |
| **UART** | 8 |
| **I2C** | 3 |
| **Ethernet** | 10/100 Mbit |
| **micro-ROS** | Частичная (community, не официальная) |
| **Цена** | ~$30-35 |

**Сервы напрямую:** ЛУЧШИЙ. До 32 серв через IntervalTimer без внешних драйверов. FlexPWM — мощнейший PWM на рынке MCU.

**Минусы:** Дорого ($35 vs $5 ESP32). micro-ROS — community-level, не официальная поддержка. Один источник (PJRC), нет альтернативных производителей.

**Итого:** Избыточно мощный для данного проекта. Оправдан если нужно 20+ серв напрямую без PCA9685 и важна скорость 600 MHz.

### Arduino Mega 2560

| Параметр | Значение |
|----------|---------|
| **Ядро** | ATmega2560, 16 MHz |
| **RAM** | 8 KB |
| **Flash** | 256 KB |
| **PWM** | 15 каналов (софтверный Servo.h — до 48) |
| **UART** | 4 |
| **micro-ROS** | Нет (только rosserial — устаревший, ROS1) |
| **Цена** | ~$10-15 |

**Вердикт: НЕ РАССМАТРИВАТЬ.** 16 MHz / 8 KB RAM / нет micro-ROS. В 2026 году это антиквариат. За те же деньги STM32 в 10 раз мощнее.

## Архитектуры для «много серв»

### Архитектура 1: ESP32-S3 + PCA9685 (бюджет)

```
Jetson Orin Nano (Уровень 2)
  │ Wi-Fi или UART
  └── ESP32-S3 (micro-ROS node)
        │ I2C
        ├── PCA9685 #1 → 16 PWM серв (ноги/колёса)
        ├── PCA9685 #2 → 16 PWM серв (манипулятор/голова)
        │ MCPWM
        ├── H-Bridge → DC мотор 1 (левое колесо)
        └── H-Bridge → DC мотор 2 (правое колесо)
```

| Плюсы | Минусы |
|-------|--------|
| Дёшево (~$12 всего) | Нет обратной связи от серв |
| Wi-Fi для отладки | Доп. плата PCA9685 на каждые 16 серв |
| micro-ROS нативно | PWM-сервы менее точные |
| OTA обновления | Нет IMU на борту |

**Стоимость:** ESP32-S3 ($5) + 2× PCA9685 ($6) + серводвигатели = **~$11 + сервы**

---

### Архитектура 2: ESP32-S3 + Bus-сервы (оптимум)

```
Jetson Orin Nano (Уровень 2)
  │ Wi-Fi или UART
  └── ESP32-S3 (micro-ROS node)
        │ UART1 (half-duplex)
        ├── LX-16A #1 ── LX-16A #2 ── ... ── LX-16A #N (шлейф, до 253 серв)
        │ MCPWM
        ├── H-Bridge → DC мотор 1
        └── H-Bridge → DC мотор 2
        │ I2C (опция)
        └── IMU (MPU6050/BNO055)
```

| Плюсы | Минусы |
|-------|--------|
| Обратная связь (позиция, температура, нагрузка) | Bus-сервы дороже ($12-30 за штуку) |
| Один провод на все сервы (шлейф) | Скорость шины ограничена (~1 Mbps) |
| Управление по ID — проще проводка | Задержка при >20 серв на одной шине |
| micro-ROS + Wi-Fi | — |

**Стоимость:** ESP32-S3 ($5) + bus-сервы = **$5 + сервы**

---

### Архитектура 3: Готовая ROS Board + Bus-сервы (комфорт)

```
Jetson Orin Nano (Уровень 2)
  │ USB
  └── YAHBOOM/Hiwonder ROS Board (STM32 на борту)
        ├── 4× encoder DC моторы (шасси)
        ├── Bus сервы (LX-16A шлейф)
        ├── 4× PWM сервы (камера pan-tilt)
        ├── IMU 9-axis (встроен)
        └── Питание для Jetson (встроенный step-down)
```

| Плюсы | Минусы |
|-------|--------|
| Всё из коробки — моторы, сервы, IMU, питание | Менее гибко (фиксированная PCB) |
| ROS2 драйверы готовы (GitHub) | STM32F103 на Yahboom слабоват (72 MHz) |
| Промышленная защита (переполюсовка, перегрев) | Дороже ($40-70 за плату) |
| Один USB к Jetson — минимум проводки | Ограничено 4 encoder моторами |
| YAHBOOM — тот же бренд что твой Orin NX | — |

**Стоимость:** плата ($40-60) + bus-сервы + DC моторы = **$40-60 + периферия**

---

### Архитектура 4: STM32 Nucleo + PCA9685 (максимум контроля)

```
Jetson Orin Nano (Уровень 2)
  │ UART/USB
  └── STM32F446RE Nucleo (micro-ROS node)
        │ Advanced Timers
        ├── 6-8 PWM серв напрямую (критичные суставы)
        │ I2C
        ├── PCA9685 → 16 PWM серв (остальные)
        │ UART1
        ├── Bus-сервы (шлейф, по необходимости)
        │ MCPWM (Advanced Timer)
        ├── FOC контроллер → BLDC мотор 1
        └── FOC контроллер → BLDC мотор 2
```

| Плюсы | Минусы |
|-------|--------|
| Максимальная прецизионность | Нет Wi-Fi (проводное подключение) |
| Прямой PWM — минимальная задержка | Сложнее в освоении (CubeMX/HAL) |
| FOC для BLDC — промышленный уровень | Нет OTA обновлений |
| 6 UART — несколько шлейфов bus-серв | Дороже ESP32 ($15-18) |

**Стоимость:** Nucleo ($15) + PCA9685 ($3) + опционально bus-сервы = **$18 + сервы**

## Итоговая рекомендация

### Для тебя: Архитектура 3 (ROS Board) + Архитектура 2 (ESP32 bus-сервы) — гибрид

Учитывая требования **«много серв, удобно, гибко»**, рекомендую **двухуровневый подход**:

```
Jetson Orin Nano (Уровень 2 — навигация)
  │
  ├── USB → YAHBOOM ROS Board V3.0 (~$45)
  │         ├── 4× DC encoder моторы (шасси/колёса)
  │         ├── 4× PWM сервы (камера pan-tilt)
  │         ├── IMU 9-axis (одометрия)
  │         └── Питание для Nano
  │
  └── UART → ESP32-S3 (~$5) [micro-ROS node]
              │ UART (bus-сервы)
              ├── Hiwonder LX-16A × N (манипулятор / шагающие ноги)
              │ I2C (опция)
              └── PCA9685 → PWM сервы (если нужны дешёвые)
```

### Почему гибрид:

| Аспект | ROS Board (шасси) | ESP32-S3 (сервы) |
|--------|-------------------|------------------|
| Задача | Колёса, базовое движение, IMU | Манипулятор, сложная кинематика |
| Сервы | 4 PWM (pan-tilt камеры) | До 253 bus-серв или 32+ PWM через PCA9685 |
| Моторы | 4 encoder DC (шасси) | — |
| Связь с Jetson | USB (надёжно) | UART или Wi-Fi |
| Масштабируемость | Фиксированная | Бесконечная (добавляй сервы на шину) |
| Стоимость | ~$45 | ~$5-11 |

### Общая стоимость электроники Уровня 4:

| Компонент | Цена |
|-----------|------|
| YAHBOOM ROS Board V3.0 | ~$45 (4к руб) |
| ESP32-S3 DevKitC-1 | ~$5 (500 руб) |
| PCA9685 × 1-2 (опция) | ~$3-6 (300-600 руб) |
| **Итого (без серв и моторов)** | **~$50-55 (5к руб)** |

Этот вариант даёт:
- **Удобство** — ROS Board из коробки, plug & play с Jetson
- **Гибкость** — ESP32 для любого количества серв (bus или PWM)
- **Масштабируемость** — добавляй сервы без замены MCU
- **Надёжность** — промышленная защита на ROS Board, отдельный MCU для серв
- **Совместимость** — YAHBOOM ROS Board + YAHBOOM Orin NX = один бренд, проверенная интеграция
