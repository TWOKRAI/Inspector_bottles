# Software Architecture — Единый фреймворк на всех уровнях

## Концепция

Использовать **multiprocess_framework** как единую платформу для приложений на каждом контроллере робота.
Каждая плата запускает своё приложение (`SystemLauncher` → процессы), а связь между платами — через ROS2 DDS.

Дополнительно — **клиентские приложения** на ПК и телефоне для мониторинга, настройки и управления.

---

## Фазы развития

Робот строится **от мозга к телу** — каждая фаза является самостоятельным полезным продуктом.

```
Фаза 1: НАСТОЛЬНЫЙ ПОМОЩНИК        Фаза 2: + ЗРЕНИЕ             Фаза 3: + ДВИЖЕНИЕ
┌──────────────────┐               ┌──────────────────┐          ┌──────────────────┐
│    Orin NX        │               │    Orin NX        │          │    Orin NX        │
│  Whisper+LLM+TTS │               │  Whisper+LLM+TTS │          │  LLM+TTS+Planner │
│  микрофон+динамик│               │  микрофон+динамик│          │  динамик          │
└──────────────────┘               ├──────────────────┤          ├──────────────────┤
                                   │    RPi 5 + HAT+  │          │    Orin Nano      │
                                   │  YOLO+лица+жесты │          │  Whisper+SLAM+Nav │
                                   │  веб-интерфейс   │          │  лидар+depth+мик  │
                                   └──────────────────┘          ├──────────────────┤
                                                                 │    RPi 5 + HAT+  │
                                                                 │  YOLO+лица+жесты │
                                                                 │  веб-интерфейс   │
                                                                 ├──────────────────┤
                                                                 │    ESP32-S3      │
                                                                 │  моторы+энкодеры │
                                                                 └──────────────────┘

Бюджет:   ~5к руб                    +~1.5к руб                    +~60-80к руб
Платы:    1 (NX)                     2 (NX + RPi)                  4 (NX+Nano+RPi+MCU)
```

**Ключевое:** на Фазе 1-2 Whisper живёт на Orin NX (GPU делится с LLM — допустимо для настольного).
При переходе к Фазе 3 Whisper **переезжает на Orin Nano** — освобождая GPU на NX полностью для LLM.

## Распределение ресурсов по платам

### Принцип распределения

Каждая плата получает **одну главную роль** и минимум побочных задач.
GPU/NPU не должен делиться между конкурирующими тяжёлыми задачами.

```
┌─── Orin NX 16GB (100 TOPS) ──── ДУМАЕТ и ГОВОРИТ ─────────────┐
│                                                                 │
│  LLMProcess          8-10 GB   GPU постоянно (inference)        │
│  PlannerProcess       0.2 GB   CPU                              │
│  FusionProcess        0.5 GB   CPU                              │
│  TTSProcess (Piper)   0.3 GB   CPU (лёгкий, не нужен GPU)      │
│  BridgeProcess        0.1 GB   CPU                              │
│  ОС + буфер          3-4 GB                                    │
│  ─────────────────────────────────────────────                  │
│  Итого: ~13 GB / 16 GB (81%)                                   │
│  GPU: только LLM — без конкуренции, максимальная скорость       │
│                                                                 │
│  Почему TTS здесь: Piper — CPU-only (~0.3 GB), и ответ LLM     │
│  уже на этой плате, не нужно гонять текст обратно.              │
└─────────────────────────────────────────────────────────────────┘

┌─── Orin Nano 8GB (40 TOPS) ──── СЛУШАЕТ и ОРИЕНТИРУЕТСЯ ───────┐
│                                                                 │
│  WhisperProcess       1.0 GB   GPU при распознавании речи       │
│  SlamProcess          2.0 GB   GPU частично (обновление карты)  │
│  NavProcess           1.0 GB   CPU (path planning)              │
│  DepthProcess         1.0 GB   GPU частично (point cloud)       │
│  OdomProcess          0.2 GB   CPU                              │
│  BridgeProcess        0.1 GB   CPU                              │
│  ОС + буфер          2.0 GB                                    │
│  ─────────────────────────────────────────────                  │
│  Итого: ~7.3 GB / 8 GB (91%)                                   │
│  GPU: SLAM + Whisper — не конфликтуют по времени                │
│  (SLAM обновляет карту периодически, Whisper — когда говорят)   │
│                                                                 │
│  Почему Whisper здесь: CUDA 40 TOPS достаточно для              │
│  whisper-medium, GPU Nano простаивает ~50% времени.             │
│  Освобождает Orin NX от конкуренции LLM vs Whisper за GPU.      │
└─────────────────────────────────────────────────────────────────┘

┌─── RPi 5 8GB + AI HAT+ (26 TOPS NPU) ── ВИДИТ и РАЗДАЁТ ─────┐
│                                                                 │
│  DetectionProcess     NPU      YOLOv8n/v9t, 30+ FPS            │
│  FaceProcess          NPU      ArcFace / SCRFD                  │
│  GestureProcess       NPU      MediaPipe / custom               │
│  TrackingProcess      0.5 GB   CPU (ByteTrack)                  │
│  WebServerProcess     0.2 GB   CPU (FastAPI + WebSocket)        │
│  BridgeProcess        0.1 GB   CPU                              │
│  ОС + буфер          3.0 GB                                    │
│  ─────────────────────────────────────────────                  │
│  Итого: ~4 GB / 8 GB (50%)                                     │
│  NPU: зрение (автономно от CPU), CPU: свободен для сервера     │
│                                                                 │
│  Почему Web-сервер здесь: CPU свободен (NPU делает всю         │
│  тяжёлую работу), Wi-Fi встроен, получает данные от всех        │
│  плат через ROS2 — идеальная точка для клиентов (ПК/телефон).  │
└─────────────────────────────────────────────────────────────────┘

┌─── Arduino Mega 2560 ──── КРУТИТ ──────────────────────────────┐
│                                                                 │
│  ШИМ для моторов (4 канала)                                     │
│  Управление сервами (8-12 каналов)                              │
│  Опрос энкодеров (прерывания)                                   │
│  Аварийный стоп (hardware watchdog)                             │
│  ─────────────────────────────────────────────                  │
│  Flash: ~30 KB / 256 KB     SRAM: ~5 KB / 8 KB                 │
│  Без фреймворка — bare metal C/C++, micro-ROS                   │
└─────────────────────────────────────────────────────────────────┘
```

## Pipeline голосового взаимодействия

```
Микрофон (на роботе)
    │
    ▼
┌─────────────────────────┐
│  Orin Nano              │
│  WhisperProcess         │
│  whisper-medium (~1 GB) │
│  Задержка: ~300 мс      │
└──────────┬──────────────┘
           │ ROS2 DDS (текст)
           │ Задержка: ~5 мс
           ▼
┌─────────────────────────┐
│  Orin NX                │
│  LLMProcess             │
│  LLaMA-3 13B Q4         │
│  Задержка: ~1-3 сек     │
└──────────┬──────────────┘
           │ внутренний IPC (текст ответа)
           ▼
┌─────────────────────────┐
│  Orin NX                │
│  TTSProcess             │
│  Piper (~0.3 GB, CPU)   │
│  Задержка: ~200 мс      │
└──────────┬──────────────┘
           │
           ▼
     Динамик (на роботе)

Общая задержка: ~1.5-3.5 сек (приемлемо для диалога)
```

**Почему не Whisper на Orin NX:**
- LLM 13B занимает 8-10 GB VRAM и GPU почти постоянно
- Whisper + LLM на одном GPU = взаимное вытеснение, задержки растут
- На Nano GPU свободен 50% времени — Whisper не мешает SLAM

**Почему whisper-medium, а не large:**
- medium (769M параметров, ~1 GB) vs large (1550M, ~1.5 GB)
- Качество medium достаточно для голосовых команд робота
- Робот распознаёт «поверни налево», а не субтитры к фильму

## Протокол связи: робот ↔ клиенты

| Канал | Протокол | Для чего |
|-------|---------|---------|
| Робот ↔ Робот (между платами) | ROS2 DDS (Ethernet) | Основная шина, real-time |

| Робот ↔ Облако (будущее) | HTTPS / MQTT | Телеметрия, обновления |

## Python + C++ — где что

| Python (основной язык) | C++ (критичные места) |
|------------------------|----------------------|
| Оркестрация, lifecycle, конфиг | Real-time control loop (PID, моторы) |
| GUI (PySide6 на ПК) | SLAM inner loop |
| LLM/STT/TTS обёртки | Image processing pipeline (tight loop) |
| Plugin-логика, chain pipeline | Hailo inference wrapper |
| ROS2 nodes (rclpy) | micro-ROS на Arduino (C) |
| Web-сервер (FastAPI) | Пользовательские ROS2 nodes (rclcpp) |
| Мониторинг, логи | Драйверы сенсоров (если нужна скорость) |

**Связка Python ↔ C++:**
- `pybind11` — для обёрток C++ библиотек в Python
- Shared memory — для zero-copy передачи данных (фреймы камеры)
- Subprocess — для изолированных C++ процессов через фреймворк

## Что нужно доработать в multiprocess_framework

| Доработка | Приоритет | Описание |
|-----------|----------|---------|
| **ROS2 BridgeModule** | Высокий | Новый модуль: Message ↔ ROS2 topic. Ключевая деталь для связи между платами |
| **WebSocket BridgeModule** | Высокий | Для связи с ПК и телефоном |
| **Headless mode** | Высокий | Работа без PySide6/GUI на Nano и RPi |
| **ARM/aarch64 совместимость** | Высокий | Проверить spawn/fork на Linux ARM |
| **Lightweight profiles** | Средний | Конфигурации для ограниченных ресурсов (RPi) |
| **Remote ConfigStore** | Средний | Изменение конфигов удалённо (с ПК/телефона) |
| **Remote StateStore** | Средний | Подписка на состояние удалённого узла |
| **Web Dashboard Process** | Средний | FastAPI + WebSocket сервер как процесс фреймворка |
| **C++ ProcessModule** | Низкий | Запуск C++ процессов под управлением фреймворка |

## Преимущества единого фреймворка

1. **Один язык разработки** — Python везде (кроме Arduino), C++ только где нужна скорость
2. **Единая система конфигов** — ConfigStore на всех платах, remote-настройка с ПК
3. **Единый подход к плагинам** — подключение алгоритмов одинаково на всех уровнях
4. **Единый мониторинг** — StateStore + подписки = видно состояние всей системы
5. **Переиспользование кода** — pipeline, plugins, config работают одинаково
6. **Один разработчик может всё** — не нужно учить N разных фреймворков
