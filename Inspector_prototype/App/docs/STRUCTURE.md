# Структура App — оценка и рекомендации

## Текущая структура

```
Inspector_prototype/           # корень проекта
├── main.py                    # точка входа (мультипроцессный менеджер)
├── App/                       # пакет приложения (UI + логика)
│   ├── main_app.py            # точка входа процесса App
│   ├── resource_paths.py      # пути к Resources
│   ├── Core/                  # ядро: базовые виджеты, конфиг, менеджеры, потоки
│   ├── Components/            # переиспользуемые UI-компоненты
│   ├── Registers/             # модели и менеджер регистров
│   ├── Widget/                # экранные виджеты (вкладки, панели)
│   ├── Windows/               # окна приложения
│   ├── Data/                  # данные на диске (рецепты, конфиги, логи)
│   ├── Resources/             # ресурсы UI (изображения, иконки)
│   └── docs/                  # документация
├── Multiproccesing/           # мультипроцессная обработка
└── Services/                  # сервисы (камеры и т.д.)
```

## Оценка

| Аспект | Оценка | Комментарий |
|--------|--------|-------------|
| Разделение слоёв | ✅ Хорошо | Core (логика/инфраструктура), Components/Widget (UI), Data/Resources (данные и ресурсы) разделены. |
| Именование папок | ✅ Понятно | Core, Components, Registers, Widget, Windows — привычные названия. |
| Ресурсы | ✅ Исправлено | Image заменён на Resources; пути через `get_resource_path()` — не зависят от CWD. |
| «App в App» | ⚠️ На усмотрение | Папка `App` внутри проекта даёт путь вида `Inspector_prototype/App/`. Для кода это просто имя пакета (`import App...`). |

## Про «App в App»

Сейчас: **Inspector_prototype** (проект) → **App** (папка/пакет приложения). Получается «приложение внутри проекта», а не «App внутри App» в буквальном смысле.

Если хочется убрать ощущение «App в App»:

1. **Переименовать пакет** `App` → например `inspector_app` или `ui`:
   - Плюс: нет повторения слова «App», структура вида `Inspector_prototype/inspector_app/`.
   - Минус: нужно заменить все импорты `App.` на `inspector_app.` (или выбранное имя).

2. **Оставить как есть** и считать `App` именем пакета приложения — так делают во многих проектах (например, `src/App/` или `project/app/`).

Рекомендация: если переименование не принципиально, оставить `App` и не трогать импорты. Если важна единообразная структура без «App в пути» — переименовать в `inspector_app` и обновить импорты.

## Дубликат App/App (исправлено)

Раньше в `RecipeManager` и `LoggingManager` к пути ошибочно добавлялся лишний сегмент `'App'`: при том что `base_dir` уже указывал на каталог пакета App, получалось `App/App/Data/...`. Из-за этого при первом запуске создавалась вложенная папка **App/App/** и копия данных там. В коде это исправлено: пути ведут в `App/Data/Recipes/` и `App/Data/logs/`. Папку **App/App/** можно удалить вручную — данные должны храниться только в **App/Data/**.

## Что куда класть

- **Resources/** — изображения, иконки, при необходимости QSS/темы. Путь в коде: `get_resource_path('имя_файла')`.
- **Data/** — только данные приложения (рецепты, конфиги, логи), не код.
- **Core/** — базовые классы, конфигурация приложения, менеджеры, потоки.
- **Registers/** — схемы и типы данных (Pydantic), не путать с папкой Data.
