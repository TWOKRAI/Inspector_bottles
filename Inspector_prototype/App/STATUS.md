## Что это

`App/` — самостоятельное PyQt5-приложение **App Inspector**: собственная архитектура (`Core/`, `UI/Windows/`, `UI/Components/`, `Registers/`, `Data/`), главный модуль `main_app.py`. Содержит обширную документацию (`docs/ARCHITECTURE.md`, `docs/NEW_ARCHITECTURE.md`).

## Вердикт

`App/` — **изолированный legacy-исследовательский прототип**, заморожен, не запускается, не используется. Активный прототип — `Inspector_prototype/multiprocess_prototype_v3/` (см. CLAUDE.md проекта).

## Варианты решения

### Вариант A — Удалить (`git rm -r App/`)

**Pros:**
- Минус 61 файл с PyQt5 → проект полностью на PySide6
- Меньше шума при `grep`/qex-поиске
- Чище CLAUDE.md (можно убрать любые упоминания)
- История сохраняется в git (`git log -- Inspector_prototype/App/`)

**Cons:**
- Утрата быстрого доступа к ранним архитектурным экспериментам и документации (`docs/ARCHITECTURE.md`, `docs/RECOMMENDATIONS.md`, `docs/DATA_ARCHITECTURE.md`)
- При желании вернуть — придётся `git checkout` из истории

### Вариант B — Архивировать (`git mv App/ Inspector_prototype/_archive/App/`)

**Pros:**
- Содержимое доступно глазами, не скрыто в git history
- Чёткий сигнал «не трогать» через имя `_archive/` (как и `multiprocess_prototype/`, `multiprocess_prototype_v2/`)
- Реверсивно — если понадобится вернуть, `git mv` обратно
- Требует минимума работы

**Cons:**
- Папка остаётся в дереве, всё ещё видна в `find`/`grep` (но `_archive/` уже принят на проекте как маркер)

### Вариант C — Мигрировать на PySide6 и интегрировать с v3

**Pros:**
- Не теряется ни одна строчка кода / документации
- Полностью единый стек

**Cons:**
- Большая отдельная фаза (12+ часов): миграция импортов, тестирование функциональности, разрешение архитектурных расхождений с v3
- Дублирование функциональности с v3 (registers, UI, lifecycle)
- Без пользовательской необходимости — overengineering

---

## Рекомендация

**Вариант B — Архивировать в `_archive/App/`.**

Обоснование:
- Сохраняет документацию (`docs/ARCHITECTURE.md` и др.) в дереве — иногда полезно сравнивать архитектурные решения v3 с предыдущей итерацией.
- Соответствует уже принятому проектному паттерну (`multiprocess_prototype/`, `multiprocess_prototype_v2/` помечены архивами в CLAUDE.md).
- Минимум работы (`git mv` + одна строка в CLAUDE.md «АРХИВ» таблице).
- Реверсивно, не теряется история.

Если в течение 3 месяцев `App/` так и не понадобится — следующий шаг безопасно перейдёт к **Варианту A** (удаление).
