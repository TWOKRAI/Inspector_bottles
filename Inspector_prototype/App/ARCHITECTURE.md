# App — Архитектура верхнего уровня

## Ответственность
Корневой пакет приложения. Точка входа, маршрутизация ресурсов, сборка всего приложения.

## Структура папок

| Папка / файл       | Ответственность |
|--------------------|-----------------|
| `main_app.py`      | Точка входа процесса: создаёт `WindowManager` и запускает Qt-приложение |
| `resource_paths.py`| Единое место для разрешения путей к ресурсам (`Resources/`) |
| `Core/`            | Бизнес-логика: менеджеры, базовые классы, фоновые потоки |
| `Components/`      | Переиспользуемые UI-компоненты (слайдеры, чекбоксы, таблицы, клавиатура) |
| `Widget/`          | Предметно-ориентированные вкладки-виджеты (обёртывают компоненты + менеджеры) |
| `Windows/`         | Окна приложения верхнего уровня (главное, загрузочное, нейронное, сообщения) |
| `Registers/`       | Модели данных Pydantic: регистры состояния и структуры данных камер/регионов |
| `Data/`            | Конфиги, рецепты (YAML), директория для логов и отладки |
| `Resources/`       | Статические ресурсы: иконки, изображения |
| `docs/`            | Документация архитектуры и решений |

## Потоки зависимостей (упрощённо)

```
main_app.py
  └── Core/Managers/WindowManager
        ├── Windows/MainWindow
        │     ├── Core/Managers/*       (DataManager, LoggingManager, ConverterManager, RecipeManager)
        │     ├── Registers/RegistersManager   ← центральное состояние
        │     ├── Widget/*              (предметные вкладки)
        │     │     └── Components/*   (слайдеры, чекбоксы, таблицы)
        │     └── Core/Managers/SortController + ParamsManager
        ├── Windows/LoadingWindow
        ├── Windows/NeurounWindow
        ├── Windows/MessageWindow
        └── Core/Threads/*             (UpdateImage, Loading, BotThread)
```

## Правила

- **Только `Core/Managers/`** содержит менеджеры (классы без UI).
- **Только `Components/`** содержит базовые UI-компоненты без бизнес-логики предметной области.
- **`Widget/`** — предметные виджеты; могут использовать менеджеры, но не владеют ими.
- **`Windows/`** — окна; только сборка UI и делегирование логики менеджерам.
- **`Registers/`** — исключительно модели данных (Pydantic), без UI и без побочных эффектов.
