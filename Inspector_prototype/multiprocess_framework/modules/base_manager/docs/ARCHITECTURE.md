# Архитектура Base Manager Module

## Концепция

Base Manager Module предоставляет фундаментальные абстракции для всех менеджеров системы.

### Метафора (аналогия с организмом)

- **BaseManager** = Профессия (врач, полицейский) - основной класс с основной функциональностью
- **BaseAdapter** = Инструмент (скальпель, микроскоп) - композиция, подключается к менеджеру
- **ObservableMixin** = Телефон (дополнительный инструмент) - миксин для логирования/мониторинга
- **ManagerExtensionMixin** = Расширения (дополнительные инструменты) - миксин для расширений

## Принципы дизайна

1. **Manager - главный класс** - вся основная функциональность в менеджере
2. **Adapter - инструмент** - подключается К менеджеру через `attach_adapter()`, добавляет только уникальную функциональность
3. **Нет дублирования** - адаптер не дублирует методы менеджера, только расширяет
4. **Композиция вместо наследования** - адаптер включается в менеджер, а не наследуется
5. **Гибкость** - у менеджера может быть несколько адаптеров разных типов

## Взаимодействие компонентов

```
BaseManager
    ├── attach_adapter() → BaseAdapter
    ├── ObservableMixin (миксин)
    └── ManagerExtensionMixin (миксин)
```

## Использование в системе

Все менеджеры системы наследуются от BaseManager:
- WorkerManager
- LoggerManager
- RouterManager
- CommandManager
- ConfigManager
- DispatchManager
