# Критерии готовности модулей

Этот документ определяет критерии, которым должен соответствовать модуль, чтобы считаться готовым к использованию.

## Обязательные критерии

### 1. Тесты (95%+ успешность)

- [ ] Все тесты проходят успешно
- [ ] Покрытие кода >80%
- [ ] Нет ошибок импорта в тестах
- [ ] Есть юнит-тесты для всех публичных методов
- [ ] Есть интеграционные тесты для взаимодействия с другими модулями

**Проверка:**
```bash
pytest modules/{module_name}/tests -v --cov=modules/{module_name} --cov-report=term-missing
```

### 2. Интерфейсы соответствуют реализации

- [ ] Все методы из интерфейсов реализованы
- [ ] Сигнатуры методов совпадают
- [ ] Типы возвращаемых значений совпадают
- [ ] Нет дополнительных обязательных параметров

**Проверка:**
```bash
python -m multiprocess_framework.refactored.tools.validate_all_modules {module_name}
```

### 3. Документация полная и актуальна

- [ ] Есть README.md с описанием модуля
- [ ] Все публичные классы имеют docstrings
- [ ] Все публичные методы имеют docstrings
- [ ] Есть примеры использования
- [ ] Документация соответствует коду

**Проверка:** Ручная проверка + валидатор

### 4. Нет зависимостей от старого кода

- [ ] Нет импортов из `multiprocess_framework.modules.*`
- [ ] Все импорты используют относительные пути
- [ ] Модуль полностью независим от старого кода

**Проверка:**
```bash
python -m multiprocess_framework.refactored.tools.validate_all_modules {module_name}
```

### 5. Примеры использования есть

- [ ] В README.md есть пример создания модуля
- [ ] Есть примеры основных сценариев использования
- [ ] Примеры работают и актуальны

**Проверка:** Ручная проверка

### 6. Интеграционные тесты есть

- [ ] Есть тесты взаимодействия с другими модулями
- [ ] Тесты проверяют основные сценарии интеграции
- [ ] Тесты проходят успешно

**Проверка:**
```bash
pytest tests/integration/ -v -k {module_name}
```

### 7. Соответствие стандартам кода

- [ ] Код следует PEP 8
- [ ] Все публичные методы имеют type hints
- [ ] Нет неиспользуемого кода
- [ ] Обработка ошибок реализована корректно

**Проверка:** Ручная проверка + линтеры

## Дополнительные критерии (желательные)

### 8. Производительность

- [ ] Нет очевидных проблем с производительностью
- [ ] Используются эффективные структуры данных
- [ ] Нет утечек памяти

### 9. Расширяемость

- [ ] Модуль легко расширяется
- [ ] Есть точки расширения (плагины, адаптеры)
- [ ] Документация описывает как расширять модуль

### 10. Обработка ошибок

- [ ] Все ошибки обрабатываются корректно
- [ ] Есть логирование ошибок
- [ ] Есть восстановление после ошибок (где применимо)

## Процесс проверки готовности

1. **Автоматическая проверка:**
   ```bash
   # Валидация модуля
   python -m multiprocess_framework.refactored.tools.validate_all_modules {module_name}
   
   # Запуск тестов
   pytest modules/{module_name}/tests -v
   
   # Проверка покрытия
   pytest modules/{module_name}/tests --cov=modules/{module_name} --cov-report=html
   ```

2. **Ручная проверка:**
   - Проверка документации
   - Проверка примеров использования
   - Проверка качества кода

3. **Результат:**
   - Если все критерии выполнены → модуль готов ✅
   - Если есть проблемы → исправить и повторить проверку

## Статусы модулей

- ✅ **READY** — модуль готов к использованию (все критерии выполнены)
- ⚠️ **IN_PROGRESS** — модуль в разработке (не все критерии выполнены)
- ❌ **NEEDS_FIXES** — модуль требует исправлений (есть критические проблемы)

## Примеры

### Модуль готов ✅

```
✅ Тесты: 100% успешность, покрытие 85%
✅ Интерфейсы: соответствуют реализации
✅ Документация: полная и актуальна
✅ Зависимости: нет зависимостей от старого кода
✅ Примеры: есть и работают
✅ Интеграция: есть тесты
✅ Стандарты: соответствуют
```

### Модуль требует исправлений ❌

```
❌ Тесты: 60% успешность, есть ошибки импорта
✅ Интерфейсы: соответствуют реализации
⚠️ Документация: неполная
✅ Зависимости: нет зависимостей от старого кода
❌ Примеры: отсутствуют
❌ Интеграция: нет тестов
⚠️ Стандарты: частично соответствуют
```

---

**Примечание:** Модуль считается готовым только если выполнены ВСЕ обязательные критерии (1-7).
