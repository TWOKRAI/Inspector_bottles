# Руководство по Рефакторингу

## 🎯 Философия Рефакторинга

Рефакторинг - это не переписывание кода с нуля, а **улучшение существующего кода** с сохранением функциональности.

### Принципы

1. **Работающий код важнее идеального**
   - Не ломай то, что работает
   - Улучшай постепенно
   - Тестируй после каждого изменения

2. **Понятность важнее краткости**
   - Код должен быть понятным
   - Лучше больше строк, но понятных
   - Комментарии помогают, но код должен быть самодокументируемым

3. **Простота важнее сложности**
   - Простые решения лучше сложных
   - Избегай преждевременной оптимизации
   - Упрощай когда возможно

### Шаг 4: Рефакторинг

#### 4.1 Упрости Структуру

**Проблема:** Слишком много уровней вложенности

**Решение:**
```python
# Было:
modules/module_name/core/submodule/deep/nested/file.py

# Стало:
modules/module_name/submodule/file.py
```

**Действия:**
1. Объедини похожие файлы
2. Удали ненужные уровни
3. Переименуй для ясности

#### 4.2 Стандартизируй Код

**Проблема:** Разные стили в разных местах

**Решение:**
1. Используй единый стиль именования
2. Используй одинаковые паттерны
3. Следуй PEP 8

**Пример:**
```python
# Было:
def get_config(self):
    ...

def retrieveConfig(self):
    ...

# Стало:
def get_config(self):
    ...

def get_config_by_name(self, name: str):
    ...
```

#### 4.3 Улучши Код

**Проблема:** Дублирование, сложные функции

**Решение:**
1. Вынеси общую логику в функции
2. Разбей большие функции на маленькие
3. Улучши обработку ошибок

**Пример:**
```python
# Было:
def process_data(self, data):
    if isinstance(data, dict):
        if 'key1' in data:
            value1 = data['key1']
            if value1 is not None:
                # ... 50 строк кода
    elif isinstance(data, list):
        # ... еще 50 строк похожего кода

# Стало:
def process_data(self, data):
    if isinstance(data, dict):
        return self._process_dict(data)
    elif isinstance(data, list):
        return self._process_list(data)
    else:
        raise ValueError(f"Unsupported data type: {type(data)}")

def _process_dict(self, data: dict):
    # ... логика обработки словаря

def _process_list(self, data: list):
    # ... логика обработки списка
```

#### 4.4 Улучши Документацию

**Проблема:** Недостаточная документация

**Решение:**
1. Добавь docstrings ко всем публичным методам
2. Обнови README
3. Добавь примеры использования

**Пример:**
```python
# Было:
def process(self, data):
    # Обрабатывает данные
    ...

# Стало:
def process(self, data: Dict[str, Any]) -> Dict[str, Any]:
    """
    Обрабатывает данные и возвращает результат.
    
    Args:
        data: Словарь с данными для обработки. Должен содержать ключи:
            - 'input': входные данные
            - 'options': опции обработки (опционально)
    
    Returns:
        Словарь с результатами обработки:
            - 'result': результат обработки
            - 'status': статус обработки ('success' или 'error')
            - 'error': сообщение об ошибке (если есть)
    
    Raises:
        ValueError: Если данные некорректны
        ProcessingError: Если произошла ошибка обработки
    
    Example:
        >>> processor = DataProcessor()
        >>> result = processor.process({'input': 'test', 'options': {}})
        >>> print(result['status'])
        'success'
    """
    ...
```

## 🔍 Типичные Проблемы и Решения

### Проблема 1: Слишком Много Уровней Вложенности

**Симптомы:**
- `modules/module/core/sub/deep/nested/file.py`
- Сложно найти нужный файл
- Непонятно где что находится

**Решение:**
- Объедини похожие файлы
- Удали ненужные уровни
- Используй плоскую структуру где возможно

### Проблема 2: Дублирование Кода

**Симптомы:**
- Похожий код в разных местах
- Изменения нужно делать в нескольких местах
- Легко забыть обновить все места

**Решение:**
- Вынеси общую логику в функции
- Создай базовые классы
- Используй композицию вместо дублирования

### Проблема 3: Сложные Функции

**Симптомы:**
- Функции >50 строк
- Много вложенных if/for
- Сложно понять что делает функция

**Решение:**
- Разбей на маленькие функции
- Вынеси логику в отдельные методы
- Используй ранний return

### Проблема 4: Непонятные Имена

**Симптомы:**
- `proc`, `mgr`, `hdlr` вместо полных имен
- Непонятно что делает функция
- Нужны комментарии чтобы понять

**Решение:**
- Используй полные имена
- Имена должны описывать назначение
- Избегай аббревиатур

### Проблема 5: Отсутствие Документации

**Симптомы:**
- Нет docstrings
- Нет примеров
- Непонятно как использовать

**Решение:**
- Добавь docstrings ко всем публичным методам
- Добавь примеры использования
- Обнови README
