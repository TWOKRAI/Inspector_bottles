## 🎯 Ключевые Концепции

### Метафора Организма

| Компонент | Организм | Роль |
|-----------|----------|------|
| **ProcessManagerCore** | Сверхэго (Мозг) | Высший уровень управления |
| **ProcessModule** | Эго (Сердце) | Базовый процесс |
| **WorkerManager** | Ид (Мышцы) | Потоки выполнения |
| **RouterModule** | Спинной мозг | Нервная система |
| **ConfigModule** | ДНК (Базовая) | Генетический код |
| **DataSchemaModule** | ДНК (Расширенная) | Структура данных |
| **MessageModule** | Мысли/Сигналы | Информационные импульсы |
| **SharedResourcesModule** | Память и Архив | Долговременная память |
| **LoggerModule** | Система мониторинга | Отслеживание жизнедеятельности |
| **CommandModule** | Исполнительная система | Выполнение команд |
| **DispatchModule** | Рефлексы | Автоматическая маршрутизация |
| **ConsoleModule** | Интерфейс с внешним миром | Взаимодействие с пользователем |

### Архитектурные Уровни

1. **Уровень 1: Сверхэго (ProcessManagerCore)** - Высший уровень управления
2. **Уровень 2: Эго (ProcessModule)** - Базовый процесс
3. **Уровень 3: Ид (WorkerManager)** - Потоки выполнения

### Взаимодействие Компонентов

- **Threads** → через WorkerManager (Ид)
- **State** → через ProcessStateRegistry
- **Communication** → через RouterManager (Спинной мозг)
- **Config** → через ConfigManager + ProcessData (ДНК)

### SharedResources

- **ProcessStateRegistry** - состояния процессов
- **ProcessData** - данные процесса (config, custom, state)
- **EventManager** - события системы
- **QueueRegistry** - реестр очередей
- **MemoryManager** - разделенная память
