# Archive — Исторические и справочные материалы

**Организовано по категориям для лучшей навигации.**

---

## 📚 Категории архива

### 🏛️ philosophical/

Философские основы архитектуры. **Ценны для глубокого понимания**, но информация отчасти дублируется в актуальной документации.

**Файлы:**
- `ARCHITECTURE.md` — основная философская архитектура (живой организм, тройца)
- `ARCHITECTURE_EVALUATION.md` — оценка архитектурных решений
- `ARCHITECTURE_INDEX.md` — исторический индекс архитектуры
- `ARCHITECTURE_PROPOSAL_SIMPLIFIED.md` — упрощённое предложение архитектуры
- `ARCHITECTURE_REGISTERS_ROUTER.md` — специфика registers и router модулей
- `ARCHITECTURE_REVIEW.md` — обзор архитектуры с критикой

**Когда читать:** Если хочешь понять глубокие корни архитектуры (но лучше читать `ARCHITECTURE_PHILOSOPHY.md` из корня!)

---

### 📊 evaluation/

Оценки полноты и готовности модулей. **Информация устаревша**, актуальная информация в `MODULES_STATUS.md` в корне.

**Файлы:**
- `COMPLETENESS.md`, `COMPLETENESS_ANALYSIS.md` — анализ полноты на старых этапах
- `COMPLETENESS_PLAN.md` — план доработки (уже выполнен)
- `FINAL_EVALUATION.md`, `FINAL_EVALUATION_V2.md` — финальные оценки версий
- `MODULE_EVALUATION.md` — оценка модулей по этапам
- `MODULE_READINESS_CRITERIA.md` — критерии готовности

**Когда читать:** Для исторического контекста, как развивались модули.

---

### 🔧 refactoring_completed/

Планы рефакторинга (уже выполнены в Phase 8/8). **Информационного значения нет**, только исторический интерес.

**Файлы:**
- `ACTION_PLAN.md`, `IMPROVEMENT_PLAN.md` — планы улучшений
- `PLAN_DICT_AT_BOUNDARY.md` — решение про Dict at Boundary (уже реализовано)
- `PLAN_IMPLEMENTATION_SUMMARY.md` — сводка реализации
- `REFACTORING_*.md` — различные отчёты о рефакторинге

**Когда читать:** Никогда (всё уже сделано, смотри текущий код).

---

### 🧪 testing_records/

Записи о тестировании на разных этапах разработки. **Устаревши**, актуальные тесты в `modules/*/tests/`.

**Файлы:**
- `TESTING_GUIDE.md`, `TESTING.md` — старые гайды по тестированию
- `TEST_*.md`, `MODULE_TESTS_FIXES.md` — отчёты о тестировании и исправлениях
- `INTEGRATION_TESTS_PLAN.md` — план интеграционных тестов

**Когда читать:** Если интересует история багов и их исправлений.

---

### 🗑️ outdated/

Полностью устаревшие документы. **Не используй**.

**Примеры:** README.md старый, BEGINNERS_GUIDE.md старый, STATUS.md старый, MODULE_STRUCTURE.md старый, и т.д.

**Когда читать:** Не читай. Если нужна информация — найди её в актуальной документации.

## ℹ️ Примечание для нейросетевых агентов

Этот архив организован для **исторического контекста**. Если вы анализируете фреймворк:

1. ✅ Начните с **актуальной** документации в корне (`/README.md` → `/FRAMEWORK_OVERVIEW.md`)
2. ✅ Для философии читайте `/ARCHITECTURE_PHILOSOPHY.md`
3. ✅ Для понимания решений читайте `/DECISIONS.md`
4. ⚠️ **Не полагайтесь на архив** — информация там частично устаревша
5. 📍 Если нужны детали — смотрите в `modules/*/README.md` и `interfaces.py`
