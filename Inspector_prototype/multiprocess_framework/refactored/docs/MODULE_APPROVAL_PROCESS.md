# Процесс утверждения модулей

Этот документ описывает процесс утверждения модулей как "готовых" к использованию.

## Обзор процесса

Процесс утверждения модуля состоит из следующих шагов:

1. Разработчик запускает валидатор модуля
2. Исправляет все проблемы
3. Создает PR с пометкой "Module Ready"
4. Автоматическая проверка (валидатор + тесты)
5. Ручная проверка документации
6. Утверждение модуля

## Детальное описание шагов

### Шаг 1: Запуск валидатора

Разработчик запускает валидатор для своего модуля:

```bash
python -m multiprocess_framework.refactored.tools.validate_all_modules {module_name}
```

**Результат:** Отчет о валидации с перечнем проблем.

### Шаг 2: Исправление проблем

Разработчик исправляет все найденные проблемы:

- Ошибки импорта
- Проблемы с интерфейсами
- Отсутствующая документация
- Проблемы с тестами
- Другие проблемы из отчета

**Критерий готовности:** Все ошибки исправлены, предупреждения минимизированы.

### Шаг 3: Создание PR

Разработчик создает Pull Request с:

- Пометкой `[Module Ready]` в названии
- Описанием изменений
- Ссылкой на отчет валидации
- Списком выполненных критериев готовности

**Пример названия PR:**
```
[Module Ready] base_manager: готов к использованию
```

**Пример описания:**
```markdown
## Модуль: base_manager

### Выполненные критерии:
- ✅ Тесты: 100% успешность, покрытие 85%
- ✅ Интерфейсы: соответствуют реализации
- ✅ Документация: полная и актуальна
- ✅ Зависимости: нет зависимостей от старого кода
- ✅ Примеры: есть и работают
- ✅ Интеграция: есть тесты
- ✅ Стандарты: соответствуют

### Отчет валидации:
[Ссылка на отчет]

### Изменения:
- Исправлены импорты
- Добавлена документация
- Улучшены тесты
```

### Шаг 4: Автоматическая проверка

CI/CD система автоматически:

1. Запускает валидатор:
   ```bash
   python -m multiprocess_framework.refactored.tools.validate_all_modules {module_name}
   ```

2. Запускает тесты:
   ```bash
   pytest modules/{module_name}/tests -v --cov=modules/{module_name}
   ```

3. Проверяет покрытие кода (>80%)

4. Проверяет соответствие критериям готовности

**Результат:** CI/CD статус (✅ или ❌)

### Шаг 5: Ручная проверка документации

Ревьюер проверяет:

- [ ] README.md полный и понятный
- [ ] Примеры использования работают
- [ ] Документация соответствует коду
- [ ] Нет опечаток и ошибок

**Критерий:** Документация готова к использованию.

### Шаг 6: Утверждение модуля

После успешного прохождения всех проверок:

1. Ревьюер утверждает PR
2. PR мержится в основную ветку
3. Статус модуля обновляется в `MODULES_STATUS.md`:
   ```markdown
   | base_manager | ✅ READY | Все критерии выполнены |
   ```

## Критерии утверждения

Модуль может быть утвержден только если:

1. ✅ Все тесты проходят (95%+ успешность)
2. ✅ Покрытие кода >80%
3. ✅ Интерфейсы соответствуют реализации
4. ✅ Документация полная и актуальна
5. ✅ Нет зависимостей от старого кода
6. ✅ Примеры использования есть
7. ✅ Интеграционные тесты есть
8. ✅ Соответствие стандартам кода

## Отклонение модуля

Модуль может быть отклонен если:

- ❌ Есть критические ошибки в тестах
- ❌ Покрытие кода <80%
- ❌ Есть зависимости от старого кода
- ❌ Документация неполная или неактуальна
- ❌ Не соответствуют стандартам кода

**Действия при отклонении:**

1. Ревьюер оставляет комментарии с описанием проблем
2. Разработчик исправляет проблемы
3. Процесс повторяется с шага 2

## Обновление статуса модулей

После утверждения модуля необходимо обновить:

1. `MODULES_STATUS.md` — статус модуля
2. `MODULES_COMPLETION_STATUS.md` — статус завершения
3. Документацию проекта (если необходимо)

## Примеры

### Успешное утверждение

```
PR: [Module Ready] base_manager: готов к использованию
Статус CI: ✅ Все проверки пройдены
Ревью: ✅ Одобрено
Результат: Модуль утвержден, статус обновлен
```

### Отклонение с комментариями

```
PR: [Module Ready] worker_module: готов к использованию
Статус CI: ❌ Тесты не проходят
Ревью: ⚠️ Требуются исправления
Комментарии:
  - Тесты проваливаются из-за проблемы с WorkerRegistry.is_enabled
  - Покрытие кода 65% (требуется 80%+)
Результат: Модуль отклонен, требуется доработка
```

---

**Важно:** Процесс утверждения гарантирует качество модулей и их готовность к использованию в продакшене.
