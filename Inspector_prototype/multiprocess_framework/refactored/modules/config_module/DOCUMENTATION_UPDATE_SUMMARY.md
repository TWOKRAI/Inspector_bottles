### 2. Переделана документация (с полным тестированием на актуальности)

#### docs/ARCHITECTURE.md (179 строк)

**Актуальная архитектура модуля:**

- **Три слоя конфигурации** — диаграмма с data_schema_module → config_module → ConfigStore
- **Config (~160 строк)** — runtime-контейнер
  - Интерфейс: get/set/update/section/subscribe
  - Особенности: dot-notation, RLock, env-fallback
  - НЕ входит: файловый I/O, валидация, конвертация моделей
  
- **ConfigManager (~215 строк)** — менеджер конфигов
  - Интерфейс: create/get/remove/list/sync/load
  - Синхронизация с ConfigStore (Dict at Boundary)
  - Удалены: StorageManager, EventManager, metadata-система
  
- **ConfigSection** — view на часть Config
  - Изменения синхронизируются с основным конфигом
  - Тот же API что Config
  
- **Интеграция** — с data_schema_module, shared_resources_module, base_manager
- **Дизайн-решения** — Dict at Boundary, No file I/O, No validation, Env-fallback opt-in
- **Производительность** — кэширование, RLock, ленивая загрузка

#### docs/USAGE_GUIDE.md (563 строки)

**Полное руководство с примерами кода:**

1. **Config — базовое использование** (30 строк)

2. **ConfigManager** (40 строк)
   - Создание, создание/получение конфигов, удаление, lifecycle

3. **Подписка на изменения** (25 строк)

4. **Работа с секциями** (35 строк)
   - Получение, синхронизация с основным конфигом, глубокие подсекции

5. **Env-fallback** (20 строк)
   - Как работает, отключение для конкретного запроса

6. **Синхронизация через ConfigStore** (30 строк)
   - Сохранение/загрузка между процессами

7. **Примеры интеграции** (100+ строк)
