# Тесты для ProcessModule

## Структура тестов

Тесты разделены по компонентам для лучшей организации:

### 1. `test_process_core.py`
Тесты для `ProcessCore` - базовый жизненный цикл процесса:
- Инициализация ядра процесса
- Управление жизненным циклом (run, stop, should_stop)
- Регистрация очередей
- Интеграция с config_manager

### 2. `test_config_handler.py`
Тесты для `ProcessConfigHandler` - работа с конфигурацией:
- Загрузка конфигурации из ConfigManager или локального config
- Получение конфигурации менеджеров
- Обновление конфигурации
- Работа с вложенными ключами

### 3. `test_managers.py`
Тесты для `ProcessManagers` - управление менеджерами:
- Инициализация основных менеджеров
- Регистрация менеджеров и адаптеров
- Пересоздание менеджеров при изменении конфигурации
- Получение статистики

### 4. `test_communication.py`
Тесты для `ProcessCommunication` - межпроцессная коммуникация:
- Регистрация очередей в queue_registry
- Регистрация каналов в роутере
- Отправка и получение сообщений
- Broadcast сообщений
- Статистика очередей

### 5. `test_process_module.py`
Тесты для `ProcessModule` - главный класс:
- Инициализация процесса с новой архитектурой
- Работа компонентов через композицию
- Удобные свойства для доступа
- Интеграция всех компонентов
- Управление жизненным циклом
- Работа с конфигурацией

### 6. `test_imports.py`
Простые тесты структуры и импортов:
- Проверка импортов всех компонентов
- Проверка структуры классов
- Проверка наличия свойств

## Запуск тестов

### Через unittest:
```bash
python3 -m unittest tests.Test_Process_module.test_process_core -v
python3 -m unittest tests.Test_Process_module.test_config_handler -v
python3 -m unittest tests.Test_Process_module.test_managers -v
python3 -m unittest tests.Test_Process_module.test_communication -v
python3 -m unittest tests.Test_Process_module.test_process_module -v
```

### Через pytest (если установлен):
```bash
pytest tests/Test_Process_module/ -v
```

### Проверка структуры:
```bash
python3 tests/Test_Process_module/test_imports.py
```
