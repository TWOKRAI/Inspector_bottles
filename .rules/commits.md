# Правило: коммиты только по явной команде пользователя

Агенты (Claude и subagents) **не делают `git commit` автоматически**. Любой коммит — только после явной команды пользователя.

## Что считается явной командой

- «закоммить», «коммить», «зафиксируй», «commit», «сделай коммит»
- «push», «запушь», «отправь» (push подразумевает предварительный коммит, если есть незакоммиченное)
- «ship», «выпусти», «релиз»
- `/ship`, `/pipeline` (внутри пайплайна коммиты делаются автоматически на завершённых этапах — это санкционированный сценарий)
- Прямое указание в задаче: «после реализации сделай коммит» / «закоммить каждый шаг»

## Что НЕ считается командой

- «давай», «погнали», «дальше», «сделай», «поправь», «фикси»
- Любая правка кода без явного слова про коммит
- Прохождение тестов или ревью — само по себе не разрешает коммит

## Если есть сомнения

Спросить: **«Закоммитить изменения сейчас?»** перед `git add` / `git commit`.

## Текущее состояние working tree после правки

После любых изменений показать пользователю:
1. `git status --short` — список изменённых файлов
2. Краткое summary что сделано
3. Предложение: «готов закоммитить, нужна команда»

## Subagents (developer / teamlead и др.)

Если задача от Director явно содержит «закоммить после реализации» — subagent делает коммит. Иначе оставляет изменения в working tree и возвращает отчёт с git status.

Direcotor (главный агент в Claude Code) при компиляции задач для subagent: **не добавляет «закоммить»** без явной команды от пользователя.

## /pipeline исключение

В режиме `/pipeline` (полный конвейер plan → implement → test → review → ship) каждый этап завершается коммитом — это явно санкционированный сценарий через сам факт запуска `/pipeline`. Дополнительного подтверждения внутри пайплайна не требуется.

## Trailers по-прежнему обязательны

Когда коммит санкционирован — trailers `Why:` и `Layer:` (+ `Refs:`) обязательны, как было до этого правила (см. `CLAUDE.md` → «Формат commit-сообщений»). Hook `commit-msg` отклонит без них.
