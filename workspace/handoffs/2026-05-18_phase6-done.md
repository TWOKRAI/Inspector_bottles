## Pre-flight architectural pivot

Изначальный план Phase 6 предполагал «переписать RecipesTab через
`BaseTreeNavTab`». После recon manager'a выяснилось — Recipes **list-CRUD**,
не tree-навигация. `SectionSpec` принципиально неприменим. Director (user)
принял ключевое решение: вместо отмены Recipes pilot — **расширить иерархию**:

- Снизить уровень `BaseTreeNavTab` → вытащить `BaseColumnarTab` как nav-агностичную базу
- Создать параллельный `BaseListNavTab(BaseColumnarTab)` для динамики
- Перевести Phase 6 на split 6a + 6b + 6c (вместо 6a + 6b)

Обоснование: проект явно строится как «конструктор» (memory
`feedback_constructor_modularity.md`); второй consumer'a НЕ гипотетический —
по плану следуют ProcessesTab, PluginManagerTab, SystemTopology Phase 2.
Не делать `BaseListNavTab` под одного consumer'а — непоследовательно
относительно Phase 4, где `BaseTreeNavTab` создавался под единственный
Settings.

## Key decisions made

- **`set_action_widget` в Standard**: реализован реально (очистка top_actions_layout + addWidget переданного), не stub. Иначе `BaseTreeNavTab._build_ui` не разместил бы action виджет.
- **ABC для абстрактных методов** — НЕ работает с QWidget (Shiboken metaclass conflict). Используется декоративный `@abstractmethod` + docstring noting.
- **`_create_content_stack()` hook** в `BaseColumnarTab` — позволяет `BaseTreeNavTab` использовать `CurrentPageStack` для diff-scroll smart-sizing. Чистое расширение, не нарушает изоляции.
- **Слот «−1» в Recipes** — реализован как обычный item (Вариант A), без специального API в `BaseListNavTab`. Хорошо для текущего scope; если в Processes/Plugins понадобятся action items — добавлять отдельно.
- **`select_key()` НЕ вызывает `_on_nav_changed`** — из-за риска recursion. Контракт зафиксирован в docstring: `_on_nav_changed` вызывается только в ответ на user-driven навигацию (signal handler).
- **`RecipesPresenter` остаётся owner persistence.** `BaseListNavTab` про файлы ничего не знает — Presenter вызывает `tab.add_item/remove_item` после своих операций.

## What did NOT work / отложено

- **LOC `recipes/tab.py` = 185** (target плана ≤100). Reviewer APPROVED с обоснованием (Recipes-specific Cards/Table + permissions + _on_action). Не блокер, но мониторить когда Processes/Plugins переедут на `BaseListNavTab` — если LOC «специфики» будет ≤100, оригинальный target был реалистичный, и можно вернуться упростить Recipes. Если ≥150 — target изначально был занижен.
- **`_selection_guard` в `BaseListNavTab`** — поле объявлено, проверяется, но нигде не устанавливается в True. Reviewer отметил как «задел на будущее» для подкласса, который захочет защититься от recursion. Не блокер.
- **Smoke `python multiprocess_prototype/run.py`** — не запущен. Архитектурно ломаться нечему (тесты зелёные), но ручной smoke рекомендуется перед мержем в main.
